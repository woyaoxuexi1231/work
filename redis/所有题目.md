### Redis 知识点

**1. 核心概念与特性**
- 内存存储、键值数据库
- 单线程事件循环与 IO 多路复用（Redis 6.0 后引入多线程 IO）
- 持久化支持、复制、高可用与分片
- 过期时间与淘汰策略
- 发布订阅与消息队列（Stream）
- 事务与 Lua 原子化执行
- 数据类型丰富（5 种基本类型 + 扩展类型）
- RESP 协议（REdis Serialization Protocol）

**2. 数据类型与底层编码**
- 字符串（string）：int、raw、embstr
- 列表（list）：quicklist（3.2+ 统一）
- 集合（set）：intset、hashtable
- 有序集合（sorted set）：ziplist（listpack）、skiplist + dict
- 哈希（hash）：ziplist（listpack）、hashtable
- 位图（bitmap）：底层是 string 的位操作
- HyperLogLog：基数统计，底层 string
- Geo：地理位置，底层使用 sorted set
- Stream：消息流，radix tree + listpack
- 扩展数据类型（Redis Stack 模块）：JSON、TimeSeries、Bloom/Cuckoo/Count-Min Sketch、Search 索引等

**3. 通用键命令**
- 键操作：`KEYS`、`SCAN`、`EXISTS`、`TYPE`、`DEL`、`UNLINK`、`RENAME`、`COPY`
- 过期与持久化：`EXPIRE`、`PEXPIRE`、`EXPIREAT`、`TTL`、`PTTL`、`PERSIST`
- 序列化：`DUMP`、`RESTORE`
- 对象信息：`OBJECT ENCODING`、`OBJECT IDLETIME`、`OBJECT REFCOUNT`
- 迁移：`MOVE`、`MIGRATE`
- 数据库管理：`SELECT`、`SWAPDB`、`FLUSHDB`、`FLUSHALL`
- 键空间通知与过期通知

**4. 各类型核心命令**
- 字符串：`SET`、`GET`、`SETNX`、`SETEX`、`PSETEX`、`MSET`、`MGET`、`INCR`、`INCRBY`、`DECR`、`APPEND`、`GETRANGE`、`SETRANGE`、`STRLEN`
- 哈希：`HSET`、`HGET`、`HMSET`、`HMGET`、`HGETALL`、`HDEL`、`HEXISTS`、`HINCRBY`、`HKEYS`、`HVALS`、`HLEN`、`HSCAN`
- 列表：`LPUSH`、`RPUSH`、`LPOP`、`RPOP`、`LRANGE`、`LINDEX`、`LINSERT`、`LLEN`、`LREM`、`LTRIM`、`BLPOP`/`BRPOP`（阻塞弹出）
- 集合：`SADD`、`SREM`、`SISMEMBER`、`SCARD`、`SMEMBERS`、`SRANDMEMBER`、`SPOP`、`SINTER`、`SUNION`、`SDIFF`、`SSCAN`
- 有序集合：`ZADD`、`ZREM`、`ZSCORE`、`ZRANK`、`ZREVRANK`、`ZRANGE`、`ZREVRANGE`、`ZRANGEBYSCORE`、`ZCOUNT`、`ZINCRBY`、`ZUNIONSTORE`、`ZINTERSTORE`、`ZSCAN`、`ZPOPMIN`、`ZPOPMAX`、`BZPOPMIN`/`BZPOPMAX`
- Bitmaps：`SETBIT`、`GETBIT`、`BITCOUNT`、`BITPOS`、`BITOP`、`BITFIELD`
- HyperLogLog：`PFADD`、`PFCOUNT`、`PFMERGE`
- Geo：`GEOADD`、`GEOPOS`、`GEODIST`、`GEORADIUS`、`GEORADIUSBYMEMBER`、`GEOHASH`
- Stream：`XADD`、`XREAD`、`XREADGROUP`、`XACK`、`XGROUP`、`XPENDING`、`XCLAIM`、`XDEL`、`XLEN`、`XRANGE`、`XREVRANGE`、`XINFO`、`XTRIM`

**5. 持久化**
- RDB（Redis Database）：快照式持久化，`SAVE`、`BGSAVE`，配置触发条件、压缩
- AOF（Append Only File）：追加写命令，`bgrewriteaof`，持久化策略 `always`、`everysec`、`no`
- RDB 与 AOF 混用：`aof-use-rdb-preamble`
- 恢复流程与数据文件修复
- 无盘复制 `repl-diskless-sync` 与持久化关系

**6. 复制（主从）**
- 建立主从关系：`REPLICAOF` 命令或配置
- 全量复制与部分复制（PSYNC）
- 复制积压缓冲区（replication backlog）
- 复制拓扑：一主多从、树状（级联）复制
- 主从切换与只读从节点
- 复制心跳与超时检测
- 无盘复制（diskless replication）

**7. 高可用（哨兵 Sentinel）**
- Sentinel 节点与 Redis 实例协同
- 主观下线（SDOWN）与客观下线（ODOWN）
- 领导者选举与故障转移（failover）
- 哨兵配置：`sentinel monitor`、`quorum`、`down-after-milliseconds`、`failover-timeout`、`parallel-syncs`
- 客户端发现：SENTINEL sentinels、SENTINEL get-master-addr-by-name
- Sentinel 集群自身管理与信息交换
- 通知脚本

**8. 集群（Redis Cluster）**
- 分片原理：16384 个哈希槽（slot）
- 数据分布：CRC16 计算槽，`CLUSTER ADDSLOTS`、`CLUSTER SETSLOT`
- 节点间通信：Gossip 协议，集群总线
- 客户端重定向：MOVED、ASK
- 哈希标签（hash tag）强制特定键同槽
- 主从与自动故障转移（不依赖 Sentinel）
- 集群扩容与缩容：`CLUSTER MEET`、`CLUSTER REPLICATE`、槽迁移
- 集群限制：多键操作需同槽，跨槽不能用事务/Lua（除非使用 `{hash_tag}`）
- `redis-cli --cluster` 管理工具

**9. 事务**
- 命令：`MULTI`、`EXEC`、`DISCARD`、`WATCH`、`UNWATCH`
- 事务特性：命令入队、原子执行（无回滚）、编译时错误 vs 运行时错误
- 乐观锁：使用 `WATCH` 实现 CAS
- 与 Lua 脚本对比：脚本保证原子性更灵活

**10. Lua 脚本**
- `EVAL`、`EVALSHA`、`SCRIPT LOAD`、`SCRIPT EXISTS`、`SCRIPT FLUSH`、`SCRIPT KILL`
- 脚本原子性执行、阻塞性
- 脚本缓存与复制：脚本命令复制 vs 脚本效果复制
- `lua-time-limit` 防止脚本长时间运行
- 与事务结合使用场景

**11. 发布订阅（Pub/Sub）**
- 频道订阅：`SUBSCRIBE`、`UNSUBSCRIBE`、`PUBLISH`
- 模式订阅：`PSUBSCRIBE`、`PUNSUBSCRIBE`
- 消息格式与客户端协议
- Pub/Sub 与 Stream 对比（不可持久化、无消费者组）
- 应用：实时通知、WebSocket 配合

**12. Stream 消息队列**
- 消费者组（`XGROUP`）与独立消费者
- 消息 ID、时间序列、范围查询
- 消息确认（ACK）与未确认消息重投递（`XCLAIM`、`XPENDING`）
- 死信与监控
- 阻塞读取与多流消费
- 消息裁剪（`XTRIM`、`MAXLEN`）与长度限制
- 相比传统消息队列的特性

**13. 管道（Pipeline）与批处理**
- 管道减少 RTT，批量发送命令
- 客户端实现（非原子，仅减少往返）
- 与事务对比
- 批量插入优化：`redis-cli --pipe`

**14. 内存管理与淘汰策略**
- 内存限制配置：`maxmemory`
- 过期键删除策略：惰性删除 + 定期删除
- 淘汰策略：
    - `noeviction`
    - `allkeys-lru`、`volatile-lru`
    - `allkeys-lfu`、`volatile-lfu`
    - `allkeys-random`、`volatile-random`
    - `volatile-ttl`
- LRU 与 LFU 实现近似算法
- 内存碎片整理：`activedefrag`
- 内存分析与诊断：`MEMORY` 系列命令、`INFO memory`

**15. 慢查询与监控**
- 慢查询日志：`SLOWLOG GET`、`SLOWLOG RESET`，配置 `slowlog-log-slower-than`
- 延迟监控：`LATENCY` 系列命令
- `INFO` 命令各模块信息：server、clients、memory、stats、replication、cpu、cluster、keyspace
- 客户端列表：`CLIENT LIST`、`CLIENT KILL`、`CLIENT SETNAME`、`CLIENT PAUSE`
- Redis 日志配置与级别

**16. 安全性**
- 密码认证：`requirepass`、`AUTH` 命令
- 危险命令重命名或禁用：`rename-command`
- ACL（Redis 6.0+）：用户管理、命令类别、键权限、频道权限
- 网络绑定与保护模式：`bind`、`protected-mode`
- 客户端连接限制与超时

**17. 性能优化与运维实践**
- 使用批量操作、管道、流水线减少往返
- 避免大键（bigkey）与热键（hotkey），诊断工具 `redis-cli --bigkeys`、`MEMORY USAGE`
- 连接池管理与复用
- 键命名规范与过期设计
- CPU、内存、网络参数调优
- 避免 `KEYS`、`MONITOR` 等阻塞命令，使用 `SCAN` 系列
- 合理使用序列化与压缩
- 监控指标与基准测试（`redis-benchmark`）

**18. 缓存设计最佳实践**
- 缓存穿透：空值缓存、布隆过滤器（RedisBloom）
- 缓存击穿：互斥锁、永不过期加逻辑过期、热点数据预热
- 缓存雪崩：过期时间随机化、多级缓存、高可用集群、降级限流
- 缓存与数据库一致性：先更新数据库再删缓存、延时双删、订阅 binlog 更新等模式
- 缓存淘汰与并发重建控制

**19. 模块与扩展（Redis Stack）**
- RedisJSON：JSON 存储与操作
- RediSearch：全文检索与二级索引
- RedisTimeSeries：时序数据
- RedisGraph：图数据库
- RedisBloom：布隆过滤器、布谷鸟过滤器等概率数据结构
- 模块加载与配置

**20. 客户端与协议**
- RESP2 与 RESP3 协议
- 连接管理、命令嵌入管道
- 客户端切换（DB、验证）
- 服务器辅助的客户端缓存（Client side caching，Redis 6.0+）
