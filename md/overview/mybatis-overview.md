## 每日一句话概览
- 第1天：在 Spring Boot 中配置 MyBatis，连接数据库，执行第一条 SQL 并返回结果。
- 第2天：编写 XML 映射文件，实现用户表的增删改查，熟悉 namespace 与 id。
- 第3天：使用注解替代 XML，完成相同 CRUD，对比两种风格差异。
- 第4天：掌握参数传递的各种方式，理解 #{} 与 ${} 的表现区别。
- 第5天：配置 resultMap 完成简单映射，使用 association 查询用户及其身份证信息。
- 第6天：使用嵌套查询与 collection 实现用户到多张订单的一对多查询。
- 第7天：编写动态条件查询接口，通过 if/choose/where 实现多种组合过滤。
- 第8天：使用 foreach 实现批量插入与批量删除，理解 collection 属性与 item。
- 第9天：引入 PageHelper 依赖，对查询结果进行自动物理分页，添加分页参数。
- 第10天：集成 MyBatis-Plus，使用 BaseMapper 获得内置 CRUD，体验 LambdaQueryWrapper。
- 第11天：配置 useGeneratedKeys 或 selectKey 获取插入后的自增主键值。
- 第12天：自定义 TypeHandler 处理枚举或 JSON 字段，感受存储与读取时的转换。
- 第13天：通过 SqlSessionTemplate 手动获取 SqlSession，批量提交 BATCH 模式的语句。
- 第14天：替换默认连接池为 Druid，开启 Web 监控页与慢 SQL 记录。
- 第15天：实现用户订单管理系统：多条件搜索、用户订单关联查询、分页展示。
- 第16天：打印并追踪 SqlSession 到 Executor 的执行链，理解四大对象各自职责。
- 第17天：验证一级缓存命中规则，对比在同一 SqlSession 与不同 SqlSession 下的查询日志。
- 第18天：开启二级缓存，对查询结果进行缓存命中压测，观察缓存失效时机。
- 第19天：实现一个统计 SQL 执行时间的插件，理解 @Intercepts 与 invocation.proceed。
- 第20天：配置延迟加载，通过 debug 查看未加载属性的代理对象及触发 SQL 时机。
- 第21天：深入 ResultSetHandler，实验 mapUnderscoreToCamelCase 与自动映射行为。
- 第22天：跟踪自定义 TypeHandler 的 setParameter 和 getResult 调用，了解何时生效。
- 第23天：通过压测对比 #{} 预编译与 ${} 拼接的性能差异，模拟 SQL 注入。
- 第24天：配置 HikariCP 核心参数，监控连接创建、等待、回收状态。
- 第25天：分析 @Transactional 下多个 Mapper 方法调用是否共享 SqlSession 与事务。
- 第26天：分别使用 foreach 拼长 SQL、BatchExecutor、MyBatis-Plus saveBatch 插入 1 万条数据，对比耗时。
- 第27天：通过打印 SQL 和压测，比较 PageHelper 拦截器改写 SQL 与手动 LIMIT 的性能。
- 第28天：构造复杂映射，对比自动映射、手动 resultMap 与构造器映射的吞吐量。
- 第29天：复现 N+1 问题，使用即时加载的 join 查询与延迟加载优化并对比性能。
- 第30天：综合评测：组合开启缓存、插件、批处理，找出高并发下的最佳配置。
- 第31天：基于 AbstractRoutingDataSource 实现读写分离，动态路由到从库。
- 第32天：接入 ShardingSphere-JDBC，让 MyBatis 透明操作分库分表后的订单表。
- 第33天：编写插件在 SQL 执行前自动追加租户 ID 条件，实现多租户隔离。
- 第34天：使用 Cursor 或 ResultHandler 流式处理百万级数据导出，避免内存溢出。
- 第35天：梳理 MyBatis/Plus/JPA 的优劣势，给出电商核心链路选型建议。
- 第36天：设计一个可根据表结构生成 Mapper、XML、Service 的代码生成器。
- 第37天：为运营后台设计动态报表方案，前端传递过滤条件，后端动态拼 SQL 并缓存。
- 第38天：设计数据加解密方案，入库时自动加密，出库时解密，对业务代码透明。
- 第39天：基于 Prometheus + Grafana 采集慢 SQL，设置阈值告警，定位问题接口。
- 第40天：为微服务设计一套注解驱动的多数据源方案，灵活切换主从或不同业务库。
- 第41天：准备面试回答：从缓存、性能、灵活度等维度对比 MyBatis 与 Hibernate。
- 第42天：准备面试回答：一二级缓存在更新、回滚、集群下的脏数据问题与解决方案。
- 第43天：准备面试回答：说明 #{} 预编译防注入机制，并指出仍可能出问题的场景。
- 第44天：准备面试回答：如何利用插件机制实现统一分页、多数据源、加密等横向能力。
- 第45天：毕业设计：设计支持分库分表、读写分离、二级缓存的订单系统持久层架构图。
