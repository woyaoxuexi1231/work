# JVM 核心面试原理清单

1. **P1: JVM 运行时数据区域划分** – 程序计数器、虚拟机栈、本地方法栈（线程私有），堆、方法区（元空间/永久代，线程共享），各区域存储内容与异常类型（StackOverflowError/OutOfMemoryError）。

2. **P2: 对象内存布局与指针压缩** – 对象头（Mark Word、Klass 指针）、实例数据、对齐填充；64 位 JVM 开启指针压缩（-XX:+UseCompressedOops）后，Klass 指针和引用从 8 字节压缩为 4 字节。

3. **P3: 类加载双亲委派机制及破坏场景** – 启动类、扩展类、应用类加载器层级；破坏场景包括 SPI（线程上下文类加载器）、Tomcat 隔离 Web 应用、OSGi 模块化，实现需重写 loadClass 或违反委派次序。

4. **P4: 类加载链接阶段（验证、准备、解析）** – 验证文件格式与字节码安全；准备为静态变量赋零值（final static 直接赋常量）；解析将符号引用转为直接引用，可配合初始化阶段延迟。

5. **P5: 栈帧结构与方法调用** – 每个栈帧含局部变量表（Slot 为单位）、操作数栈、动态链接、方法返回地址；invokevirtual 动态分派查虚表，invokestatic/invokespecial 静态分派。

6. **P6: 垃圾回收分代假说与对象生命周期** – 弱分代假说（大部分对象朝生夕灭），强分代假说（越老越难消亡）；堆分代为新生代（Eden、Survivor S0/S1）与老年代，跨代引用通过记忆集（卡表）解决。

7. **P7: 可达性分析与 GC Roots** – 以 GC Roots 为起点（栈帧局部变量、静态变量、JNI 引用、活跃线程等）遍历对象图，不可达对象被回收；区别于引用计数（无法解决循环引用）。

8. **P8: 四种引用类型（强、软、弱、虚）** – 强引用永不回收；软引用在 OOM 前回收（缓存）；弱引用下次 GC 即回收（WeakHashMap、ThreadLocal 键）；虚引用用于跟踪对象回收（DirectByteBuffer 清理）。

9. **P9: 标记-清除/复制/标记-整理算法取舍** – 标记-清除产生碎片；复制（新生代）浪费内存但快速；标记-整理（老年代）移动对象减少碎片，但停顿较长。

10. **P10: Serial/Parallel/CMS/G1 垃圾收集器演进** – Serial 单线程；Parallel 吞吐优先（自适应调优）；CMS 并发标记（初始标记、并发标记、重新标记、并发清除），浮动垃圾与碎片问题；G1 基于 Region 和停顿预测模型。

11. **P11: G1 收集器核心机制（Region、RSet、SATB）** – 堆分为多个 Region（E/S/O/H），RSet 记录跨 Region 引用；并发标记阶段使用快照（SATB）避免漏标，最终通过重新标记和清理；Mixed GC 逐步回收老年代 Region。

12. **P12: ZGC 染色指针与读屏障** – 指针高 4 位存放染色状态（Finalizable、Remapped、Marked0、Marked1），无需维护 RSet；读屏障在对象引用加载时检查并修正指针，实现并发整理。

13. **P13: 安全点（SafePoint）与安全区域（SafeRegion）** – GC 等全局操作需线程到达安全点（方法调用、循环回边等）；长时间阻塞的线程（如 JNI 临界区）进入安全区域，离开时检查全局标志。

14. **P14: 记忆集与卡表（Card Table）** – 记忆集记录非收集区域指向收集区域的引用；卡表是记忆集的一种实现，字节数组每个元素对应一块内存（512 字节），脏卡标识跨代引用。

15. **P15: 逃逸分析及其优化（栈上分配、标量替换、锁消除）** – 分析对象是否逃逸出线程或方法；未逃逸对象可栈上分配（免 GC）、标量替换为局部变量、同步锁消除（无竞争）。

16. **P16: 方法内联与编译器优化阈值** – 热点方法编译时将被调用方法字节码嵌入调用者，消除调用开销；内联阈值由 -XX:MaxInlineSize（字节数）和调用次数（热点计数）控制。

17. **P17: 分层编译（C1、C2 协同）** – 0 层解释执行，1 层 C1 带 profiling，2 层 C1 无 profiling，3 层 C1 全 profiling，4 层 C2 优化编译；根据方法调用/循环计数在层间切换，平衡启动与峰值性能。

18. **P18: synchronized 锁升级与对象头 Mark Word** – 偏向锁记录线程 ID（CAS 置入），轻量锁在当前栈帧分配 Lock Record 并 CAS 替换 Mark Word，重量锁指向操作系统的互斥量；Mark Word 存储锁信息、哈希码、分代年龄。

19. **P19: volatile 内存屏障实现** – 写操作后插入 StoreStore+StoreLoad 屏障（强制刷新到主存），读操作前插入 LoadLoad+LoadStore 屏障（从主存读）；避免指令重排，不保证原子性。

20. **P20: JVM 性能监控工具与转储分析** – jstat 查看 GC 和编译统计；jmap 导出堆转储（heap dump）；MAT 分析内存泄漏（支配树、浅/深堆、GC Root 路径）；jstack 死锁检测与线程状态。
