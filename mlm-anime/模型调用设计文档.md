# 模型调用设计文档

## 1. 概述

本项目需要统一调用火山引擎和 DeepSeek 的模型 API，涵盖三种模型类型：文本、图片、视频。核心差异在于：**文本和图片是同步调用**（发请求 → 拿结果），**视频是异步调用**（创建任务 → 轮询查询状态 → 拿结果）。

因此设计一张统一的 `model_call_task` 表，记录每次调用的完整生命周期。

---

## 2. 数据库设计

### 2.1 模型调用任务表：`model_call_task`

| 字段 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| `id` | BIGINT | ✓ | 主键，自增 |
| `model_id` | VARCHAR(128) | ✓ | 调用的模型 ID，对应 `model` 表的 `model_id` |
| `model_type` | VARCHAR(16) | ✓ | 模型类型：`TEXT` / `IMAGE` / `VIDEO` |
| `provider` | VARCHAR(32) | ✓ | 提供商：`VOLCENGINE` / `DEEPSEEK` |
| `status` | VARCHAR(16) | ✓ | 任务状态（见 2.2） |
| `external_task_id` | VARCHAR(128) | | 外部任务 ID，视频模型必用（火山返回的 `id`，用于轮询查询） |
| `request_params` | TEXT | | 请求参数，JSON 格式，记录本次调用的完整请求体 |
| `response_result` | MEDIUMTEXT | | 原始响应结果，JSON 格式 |
| `output_url` | VARCHAR(1024) | | 输出文件 URL（图片 URL / 视频 URL），有效期 24 小时 |
| `error_message` | VARCHAR(1024) | | 错误信息 |
| `created_at` | DATETIME | ✓ | 创建时间 |
| `updated_at` | DATETIME | ✓ | 更新时间 |

### 2.2 `status` 状态流转

```
PENDING ──→ PROCESSING ──→ SUCCESS
   │             │
   └─────────────┴──→ FAILED
```

| 状态 | 说明 | 适用类型 |
|------|------|------|
| `PENDING` | 任务已创建，待提交 | 全部 |
| `PROCESSING` | 执行中（文本/图片瞬间完成；视频在此期间轮询） | 全部 |
| `SUCCESS` | 执行成功，结果已写入 `output_url` 或 `response_result` | 全部 |
| `FAILED` | 执行失败，原因写入 `error_message` | 全部 |

- **文本/图片**：PENDING → PROCESSING → SUCCESS/FAILED（毫秒级完成）
- **视频**：PENDING → PROCESSING（接到 task_id，开始轮询）→ SUCCESS/FAILED（可能几分钟）

### 2.3 DDL

```sql
CREATE TABLE `model_call_task` (
    `id`               BIGINT        NOT NULL AUTO_INCREMENT COMMENT '主键',
    `model_id`         VARCHAR(128)  NOT NULL COMMENT '调用的模型 ID',
    `model_type`       VARCHAR(16)   NOT NULL COMMENT '模型类型：TEXT / IMAGE / VIDEO',
    `provider`         VARCHAR(32)   NOT NULL COMMENT '提供商：VOLCENGINE / DEEPSEEK',
    `status`           VARCHAR(16)   NOT NULL DEFAULT 'PENDING' COMMENT '状态：PENDING / PROCESSING / SUCCESS / FAILED',
    `external_task_id` VARCHAR(128)  NULL COMMENT '外部任务 ID（视频模型使用）',
    `request_params`   TEXT          NULL COMMENT '请求参数（JSON）',
    `response_result`  MEDIUMTEXT    NULL COMMENT '原始响应结果（JSON）',
    `output_url`       VARCHAR(1024) NULL COMMENT '输出文件 URL（图片/视频）',
    `error_message`    VARCHAR(1024) NULL COMMENT '错误信息',
    `created_at`       DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updated_at`       DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`),
    KEY `idx_model_id` (`model_id`),
    KEY `idx_status` (`status`),
    KEY `idx_model_type_status` (`model_type`, `status`),
    KEY `idx_external_task_id` (`external_task_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='模型调用任务表';
```

## 3. 各类型 API 调用流程

### 3.1 文本模型（同步）

```
用户请求
  │
  ├── 火山引擎: POST https://ark.cn-beijing.volces.com/api/v3/responses
  │     request: { model, input, thinking, ... }
  │     response: { id, output: [{ content: [{ text: "..." }] }], ... }
  │
  └── DeepSeek:  POST https://api.deepseek.com/chat/completions
        request: { model, messages, thinking, ... }
        response: { choices: [{ message: { content: "..." } }], ... }
```

- 发请求 → 拿到结果 → 记录到 `response_result` → 状态置为 `SUCCESS`
- 火山引擎返回的 `response.id` 可保存，用于多轮对话

### 3.2 图片模型（同步）

```
POST https://ark.cn-beijing.volces.com/api/v3/images/generations
  request: { model, prompt, image?, size?, ... }
  response: { data: [{ url: "..." }], ... }
```

- 发请求 → 拿到图片 URL → 记录到 `output_url` → 状态置为 `SUCCESS`
- 图片 URL 有效期 24 小时，需及时转存

### 3.3 视频模型（异步）

```
Step 1: 创建任务
POST https://ark.cn-beijing.volces.com/api/v3/contents/generations/tasks
  request: { model, content: [...], resolution, duration, ... }
  response: { id: "cgt-xxx" }  ← 得到 external_task_id

Step 2: 轮询查询
GET https://ark.cn-beijing.volces.com/api/v3/contents/generations/tasks/{id}
  状态: queued → running → succeeded / failed
  成功后: { content: { video_url: "..." } }
```

- 创建任务 → 拿到 `external_task_id` → 更新状态为 `PROCESSING` → 定时轮询
- 轮询到 `succeeded` → 拿到 `video_url` → 记录到 `output_url` → 状态置为 `SUCCESS`
- 轮询到 `failed` → 记录错误 → 状态置为 `FAILED`
- 视频 URL 有效期 24 小时

### 4.1 核心类说明

#### ModelCaller（调用器接口）

```java
// 统一调用接口
public interface ModelCaller {
    // 是否支持该提供商
    boolean supports(Provider provider);
    // 调用文本模型（同步）
    CallResponse callText(CallRequest request);
    // 调用图片模型（同步）
    CallResponse callImage(CallRequest request);
    // 创建视频任务（异步 - 仅返回 task_id）
    CallResponse createVideoTask(CallRequest request);
    // 查询视频任务状态
    CallResponse queryVideoTask(String externalTaskId);
}
```

#### ModelCallService（调用编排）

```java
public class ModelCallService {
    // 统一入口：根据 model_id 查 model 表 → 确定类型和提供商 → 路由到对应 Caller
    public Long call(CallRequest request) {
        // 1. 查 model 表获取 model_type、provider
        // 2. 创建 model_call_task 记录，状态 PENDING
        // 3. 根据 model_type 分流：
        //    - TEXT/IMAGE → 同步调用 → 写结果 → 状态 SUCCESS
        //    - VIDEO → 异步创建任务 → 拿到 external_task_id → 状态 PROCESSING
        // 4. 返回 task.id
    }
}
```

#### VideoTaskPoller（视频轮询）

```java
@Component
public class VideoTaskPoller {
    // 每 10 秒扫描一次状态为 PROCESSING 的视频任务
    @Scheduled(fixedDelay = 10000)
    public void poll() {
        // 1. 查询 model_call_task WHERE model_type='VIDEO' AND status='PROCESSING'
        // 2. 逐个调用 ModelCaller.queryVideoTask(external_task_id)
        // 3. 如果 succeeded → 写 output_url → 状态 SUCCESS
        // 4. 如果 failed → 写 error_message → 状态 FAILED
        // 5. 如果 queued/running → 跳过，等下次轮询
    }
}
```

### 4.2 调用流程图

```
Controller
    │
    ▼
ModelCallService.call(request)
    │
    ├── 查 model 表 → 拿到 model_type + provider
    ├── 创建 model_call_task (status=PENDING)
    │
    ├── TEXT/IMAGE（同步）
    │   ├── 选 Caller（VolcengineCaller / DeepSeekCaller）
    │   ├── 调 API → 拿到结果
    │   ├── 写回 model_call_task (status=SUCCESS, response_result/output_url)
    │   └── 返回
    │
    └── VIDEO（异步）
        ├── VolcengineCaller.createVideoTask()
        ├── 拿到 external_task_id
        ├── 写回 model_call_task (status=PROCESSING, external_task_id)
        └── 返回（后续由 VideoTaskPoller 轮询完成）
```

## 5. API Key 管理

API Key 属于敏感配置，建议放在 `application.yml` 中，不存入数据库：

```yaml
provider:
  volcengine:
    api-key: ${VOLCENGINE_API_KEY}
  deepseek:
    api-key: ${DEEPSEEK_API_KEY}
```

环境变量注入，避免密钥泄露到代码仓库。

## 6. 补充建议

当前两张表（`model` + `model_call_task`）覆盖了模型管理和调用记录的核心场景。后续按需可以扩展：

| 方向 | 说明 |
|------|------|
| 文件转存 | 图片/视频 URL 24 小时过期，需要自动下载转存到 OSS/TOS，可在 `output_url` 旁加 `stored_url` 字段 |
| 调用统计 | 按模型/日期统计调用次数、成功率、耗时，可在 `model_call_task` 基础上做聚合查询，无需额外表 |
| 多轮对话 | 文本模型的火山 API 支持 `previous_response_id`，如需保留对话历史，可以在 task 上加 `previous_task_id` 串联 |
| 用户系统 | 当前设计未涉及用户，后续加 `user_id` 字段即可 |
