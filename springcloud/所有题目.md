以下为 **Spring Cloud**（微服务全家桶，Hoxton/2020.0.x 及之后版本体系）的核心知识点列表，聚焦 Spring Cloud 官方子项目及其抽象，不涉及 Spring Cloud Alibaba、Spring Security 等外部或安全相关生态。

---

### Spring Cloud 知识点

**1. 服务注册与发现**
- 服务发现抽象：`DiscoveryClient`
- 服务注册抽象：`ServiceRegistry`
- Eureka：服务端（Eureka Server）与客户端（Eureka Client）、自我保护模式、健康检查、分区（region/zone）、元数据、REST 端点
- Consul：服务注册、健康检查（TTL/HTTP/gRPC）、KV 存储集成、多数据中心
- Zookeeper：通过 Spring Cloud Zookeeper 实现服务注册与发现（基于临时节点）
- Eureka 集群：相互注册、同步
- 服务剔除与心跳续约机制
- `@EnableDiscoveryClient` 与 `@EnableEurekaClient` 的区别
- 服务实例状态与自定义健康检查

**2. 负载均衡**
- `Spring Cloud LoadBalancer`（官方推荐，替代 Ribbon）
- 基于 WebClient 与 `@LoadBalanced` RestTemplate 的负载均衡集成
- 负载均衡策略：轮询、随机、自定义 `ReactorLoadBalancer`
- 服务实例选择与过滤：`ServiceInstanceListSupplier`
- 缓存机制、健康检查与重试集成
- Ribbon（已进入维护模式）：IRule、IPing、ServerList 配置、饥饿加载

**3. 声明式 HTTP 客户端（OpenFeign）**
- `@FeignClient`：定义接口绑定远程服务
- 契约：SpringMVC 注解支持、Feign 原生注解、自定义 `Contract`
- 编解码：`Encoder`/`Decoder`、Spring 的 `HttpMessageConverters` 集成
- 请求拦截器：`RequestInterceptor` 实现请求头传递、鉴权信息填充
- 超时与重试配置
- 日志级别：NONE、BASIC、HEADERS、FULL
- 错误处理：`ErrorDecoder`、降级 `fallback`/`fallbackFactory`
- Hystrix/Resilience4j 集成熔断（通过 `feign.circuitbreaker.enabled`）
- 压缩设置与请求/响应 GZIP
- 动态 URL 与手动创建 Feign Client
- 与 LoadBalancer 的结合使用

**4. 服务容错**
- Spring Cloud Circuit Breaker 抽象层
- 支持实现：Resilience4j（推荐）、Hystrix（已停更，仅维护）
- 断路器核心状态：CLOSED、OPEN、HALF_OPEN
- 时间窗口、失败阈值、滑动窗口类型
- 舱壁隔离：信号量隔离、线程池隔离
- 限流：RateLimiter
- 重试机制：Retry
- 超时与异常配置
- `@CircuitBreaker` 注解使用与回调方法
- Fallback 逻辑与熔断事件监听

**5. 服务网关**
- Spring Cloud Gateway（基于 WebFlux）
- 核心组件：`Route`（路由）、`Predicate`（断言）、`Filter`（过滤器）
- 路由断言工厂：`Path`、`Host`、`Header`、`Query`、`Method`、`Cookie`、`Weight` 等
- 网关过滤器：局部（GatewayFilter）、全局（GlobalFilter）
- 内置过滤器：`AddRequestHeader`、`AddRequestParameter`、`AddResponseHeader`、`PrefixPath`、`StripPrefix`、`RewritePath`、`CircuitBreaker`、`RequestRateLimiter`、`Retry` 等
- 自定义过滤器
- 限流集成：基于 Redis 的 `RequestRateLimiter`
- 负载均衡：`lb://service-name` URI 方案
- 动态路由：结合配置中心刷新
- 跨域配置、安全头设置
- 与 Hystrix/Sentinel 的熔断结合
- Zuul 1.x（已停止维护）基本概念：路由、过滤器（pre、route、post、error）

**6. 配置中心**
- Spring Cloud Config：服务端（Config Server）与客户端（Config Client）
- 后端存储：Git、SVN、本地文件、Vault、JDBC、Consul KV、CredHub 等
- 配置刷新：`@RefreshScope`、`/actuator/refresh` 端点
- Spring Cloud Bus：基于消息代理（RabbitMQ、Kafka）实现配置批量刷新
- 自动刷新：结合 Git Webhook 与 Bus 的 `/monitor` 端点（或 `/busrefresh`）
- 配置加密解密：对称加密与非对称加密（`{cipher}` 前缀）
- 占位符与多仓库匹配：`{application}`、`{profile}`、`{label}` 路径结构
- 复合环境仓库（composite）
- 客户端快速失败与重试、健康指示器
- 通过 Vault 管理敏感配置

**7. 消息驱动（Spring Cloud Stream）**
- 编程模型：基于 Spring Messaging 的 `@Input`、`@Output`、`@StreamListener`（已废弃），函数式编程模型 `java.util.function`（Supplier、Function、Consumer）
- Binder 抽象：支持 RabbitMQ、Kafka、Kafka Streams、Partitioning 等
- 绑定配置：`spring.cloud.stream.bindings`
- 消费者组与分区、并发消费
- 消息转换：`contentType` 与自动序列化/反序列化
- 错误处理：`errorChannel`、重试、死信队列
- 消息可视化与跟踪集成
- 多 binder 支持与动态目标

**8. 分布式追踪（Spring Cloud Sleuth）**
- 与 Brave、Zipkin 集成，自动生成 TraceId 和 SpanId
- 日志关联：将 Tracing 信息注入 MDC，输出到日志
- 传播机制：通过 HTTP、消息中间件、Feign 等传递追踪上下文
- 采样策略：采样率、正则排除
- 与 Zipkin Server 配合进行链路可视化
- 与各种组件自动集成（RestTemplate、WebClient、Gateway、Stream）
- 注意：Spring Cloud Sleuth 已进入维护期，后续由 Micrometer Tracing 取代，但核心概念一致

**9. Spring Cloud Task**
- 短生命周期微服务/批处理任务支持
- 任务自动记录：开始/结束时间、退出码、外部执行参数
- 与 Spring Batch 集成，持久化任务执行记录
- 任务执行监听器、自定义任务库
- 与 Data Flow 配合编排

**10. Spring Cloud Contract**
- 消费者驱动的契约测试（CDC）
- 契约定义：Groovy DSL、YAML
- 生成存根（Stubs）并发布到仓库
- 消费者端使用 `StubRunner` 进行集成测试
- 与 WireMock 集成模拟服务端
- 消息契约：针对消息驱动的场景

**11. Spring Cloud Function**
- 函数即服务：将业务封装为 `Function`、`Consumer`、`Supplier`
- 与 Spring Cloud Stream 结合实现无关注解的函数式编程
- 多 I/O、动态路由、函数组合
- 适配 AWS Lambda、Azure Functions 等平台

**12. Spring Cloud Kubernetes**
- 服务发现：基于 Kubernetes Service 和 Endpoints
- 配置管理：读取 ConfigMap 和 Secret，动态刷新
- 负载均衡：Kubernetes Service 实现负载均衡与 Ribbon/LoadBalancer 结合
- 熔断、健康探测与命名空间支持
- `spring-cloud-starter-kubernetes-client` 使用 API 客户端

**13. 其他核心通用能力**
- `@EnableCircuitBreaker` / `@SpringCloudApplication`（不推荐使用）
- Spring Cloud 上下文：Bootstrap 上下文（已弃用，推荐使用 Spring Boot 配置方式）、`spring.config.import`
- 公共抽象：`HealthIndicator` 集成、`Environment` 定制
- 路由与过滤器编排（在 Gateway 中）
- 跨服务共享会话与上下文传递
- 版本与兼容性：Spring Cloud 与 Spring Boot 的版本对应表（Release Train）

---

以上列表完整覆盖了 Spring Cloud 官方微服务解决方案中各核心组件的知识点，严格限定于 Spring Cloud 生态本身，未纳入 Alibaba、Security 等额外项目。
