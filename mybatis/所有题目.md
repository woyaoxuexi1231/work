以下为 **MyBatis**（核心框架，版本 3.x）的完整知识点列表，仅聚焦 MyBatis 本身，不涉及 MyBatis-Spring、MyBatis Generator、MyBatis Plus 等衍生项目。

---

### MyBatis 知识点

**1. 核心概念与架构**
- ORM 与 SQL 映射思想
- 功能架构：接口层、数据处理层、框架支撑层
- 核心组件：`SqlSessionFactory`、`SqlSession`、`Configuration`
- 映射器（Mapper）接口动态代理
- 生命周期与作用域

**2. 配置文件（mybatis-config.xml）**
- 配置结构：`<properties>`、`<settings>`、`<typeAliases>`、`<typeHandlers>`、`<plugins>`、`<environments>`、`<mappers>`
- 属性（properties）：引入外部属性文件、属性占位符
- 设置（settings）：所有配置项（如 `cacheEnabled`、`lazyLoadingEnabled`、`mapUnderscoreToCamelCase` 等）
- 类型别名（typeAliases）：内置别名、`<package>` 批量注册
- 类型处理器（typeHandlers）：自定义 `TypeHandler`、继承 `BaseTypeHandler`、处理 JdbcType 与 JavaType
- 环境配置（environments）：多环境支持，事务管理器（`JDBC`、`MANAGED`），数据源（`POOLED`、`UNPOOLED`、`JNDI`）
- 映射器（mappers）：配置方式（resource、class、package、url）
- 插件（plugins）：拦截器原理与配置
- 对象工厂（objectFactory）：自定义 `ObjectFactory`
- 配置加载流程与合并规则

**3. SQL 映射文件（Mapper XML）**
- 顶级元素：`<cache>`、`<cache-ref>`、`<resultMap>`、`<sql>`、`<insert>`、`<update>`、`<delete>`、`<select>`
- `<select>`：属性（`id`、`parameterType`、`resultType`、`resultMap`、`flushCache`、`useCache`、`timeout` 等）
- `<insert>`：自动生成主键（`useGeneratedKeys`、`keyProperty`）、`selectKey` 子元素
- `<update>` 和 `<delete>`：基本用法
- 参数传递：`#{}` 占位符（预编译安全）与 `${}` 字符串替换（非安全）
- 参数类型：简单类型、Map、JavaBean、注解 `@Param`
- 结果映射（`resultMap`）：`<id>`、`<result>`、`<association>`、`<collection>`、`<discriminator>`
- 高级关联：一对一级联、一对多集合、鉴别器根据值选择映射
- 延迟加载：全局配置 `lazyLoadingEnabled`、局部 `fetchType`
- `<sql>` 复用片段与 `<include>`
- 命名空间：绑定 Mapper 接口全限定名
- 列名与属性映射规则，`autoMappingBehavior`
- 构造方法映射：`<constructor>`、`<arg>`、`<idArg>`

**4. 注解映射（Mapper 注解）**
- 常用注解：`@Select`、`@Insert`、`@Update`、`@Delete`
- `@Results`、`@Result`、`@One`、`@Many`、`@ResultMap`
- 动态 SQL 注解：`@SelectProvider`、`@InsertProvider`、`@UpdateProvider`、`@DeleteProvider`
- 注解参数：`@Param`、`@Options`（主键设置等）
- `@ConstructorArgs`、`@Arg` 构造注入
- `@CacheNamespace`、`@CacheNamespaceRef` 缓存配置
- 注解与 XML 组合方式

**5. 动态 SQL**
- `<if>`
- `<choose>`、`<when>`、`<otherwise>`
- `<where>`：智能去除 AND/OR 前缀
- `<set>`：智能去除逗号后缀（用于 UPDATE）
- `<trim>`：`prefix`、`suffix`、`prefixOverrides`、`suffixOverrides`
- `<foreach>`：迭代集合/数组，属性 `collection`、`item`、`index`、`open`、`close`、`separator`
- `<bind>`：将表达式绑定为变量
- `<script>`：在注解中支持动态 SQL 片段
- 多参数处理与 OGNL 表达式

**6. SQL 语句构建器类**
- `SQL` 类（Java 链式 API）
- 方法：`SELECT()`、`FROM()`、`WHERE()`、`JOIN()`、`INNER_JOIN()`、`LEFT_OUTER_JOIN()`、`RIGHT_OUTER_JOIN()`
- 动态构建 `INSERT()`、`UPDATE()`、`DELETE()`
- 使用构建器结合 `@SelectProvider` 等实现动态 SQL 提供器

**7. Java API（SqlSession 操作）**
- `SqlSessionFactoryBuilder` 构建 `SqlSessionFactory`
- `SqlSessionFactory`：`openSession()`、`openSession(boolean autoCommit)`、`openSession(Connection)`、`openSession(TransactionIsolationLevel)`
- `SqlSession` 主要方法：
    - 查询：`selectOne()`、`selectList()`、`selectMap()`、`selectCursor()`（游标）
    - 插入：`insert()`
    - 更新：`update()`
    - 删除：`delete()`
    - 事务：`commit()`、`rollback()`、`close()`
- 获取 Mapper：`getMapper(Class<T>)`
- 配置对象获取：`getConfiguration()`
- 批量执行：`SqlSession` 的 `BatchExecutor`
- 分页插件原理：分页与 `RowBounds` 参数（逻辑分页）

**8. 缓存机制**
- 一级缓存（本地缓存）：`SqlSession` 级别，默认开启，无法关闭（可清理）
- 二级缓存：`namespace` 级别，需 `<cache/>` 启用，可自定义缓存实现（实现 `Cache` 接口）
- `<cache>` 属性：`eviction`（LRU、FIFO、SOFT、WEAK）、`flushInterval`、`size`、`readOnly`
- `<cache-ref>` 引用其他命名空间缓存
- `useCache` 和 `flushCache` 控制单次查询缓存行为
- 缓存的序列化要求（读写属性与 `Serializable`）
- 与第三方缓存集成（如 Ehcache、Redis，通过适配器，但仅概念）

**9. 日志**
- MyBatis 日志工厂自动选择：SLF4J、Log4j2、Log4j、JDK Logging、Commons Logging
- 日志名称：对应命名空间
- 配置日志实现：添加对应 jar 并设置日志框架
- 打印完整 SQL 语句与参数

**10. 插件（拦截器）**
- `Interceptor` 接口与 `Intercepts`、`Signature` 注解
- 拦截对象与接口：`Executor`（update、query、commit、rollback）、`ParameterHandler`（setParameters）、`ResultSetHandler`（handleResultSets）、`StatementHandler`（prepare、parameterize、batch、update、query）
- 插件链与动态代理创建（`Plugin` 类）
- 分页插件等典型应用原理
- 多插件顺序

**11. 高级类型处理**
- 类型处理器（TypeHandler）：映射 JDBC 类型与 Java 类型
- 自定义 TypeHandler 实现：`setParameter`、`getResult`（通过列名、索引、存储过程）
- 枚举映射：`EnumTypeHandler`、`EnumOrdinalTypeHandler`、自定义
- `@MappedJdbcTypes`、`@MappedTypes` 注解
- 泛型类型处理

**12. 结果集处理细节**
- 自动映射行为：`PARTIAL`、`NONE`、`FULL`
- 列名与属性下划线转驼峰自动映射 `mapUnderscoreToCamelCase`
- 结果集映射：多结果集支持
- `resultOrdered` 属性
- 鉴别器映射（`<discriminator>`）
- 关联的嵌套 Select 查询与嵌套结果映射（`<association>` 的 `select` 属性与 `<collection>` 的 `select` 属性）
- 延迟加载触发条件

**13. 与数据库交互**
- 预编译语句与参数占位符
- 获取自增主键：`useGeneratedKeys` 与 `keyProperty`，或 `<selectKey>`
- 批量操作：与 `BatchExecutor` 配合或多次操作
- 存储过程调用：`<select>`、`<update>` 等中声明 `statementType="CALLABLE"`，使用 JDBC 参数
- 多参数支持 `Map`、`@Param`、JavaBean
- `ResultHandler` 自定义处理每行结果
- 游标 `Cursor<T>`：流式处理大量数据

**14. 工具与扩展（MyBatis 核心内）**
- 代码生成元数据（由配置解析）
- 资源加载与类加载
- 脚本语言驱动：默认 OGNL，可扩展（`LanguageDriver`）
- `VFS` 虚拟文件系统抽象
- `Reflector` 与反射工具集

---

以上列表完整覆盖了 MyBatis 核心框架自身的所有主要知识点，未涉及任何与 Spring 集成或第三方增强库的内容。
