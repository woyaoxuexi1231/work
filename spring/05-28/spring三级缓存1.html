什么是循环依赖？

简单来说，就是 A 依赖 B，同时 B 又依赖 A。 当 Spring 创建 A 时，发现需要 B；去创建 B 时，发现又需要 A。这就陷入了死循环。

public class A {
  @Autowired
  private B b;
}

public class B {
  @Autowired
  private A a;
}

三级缓存的定义

  • 一级
    singletonObjects (单例池)

    存放完全初始化好、可以直接使用的 Bean。

  • 二级
    earlySingletonObjects (早期单例池)

    存放原始的 Bean 对象（尚未填充属性），用于解决循环引用。

  • 三级
    singletonFactories (单例工厂池)

    存放 ObjectFactory，用于在需要时生成早期 Bean 引用（可能是代理对象）。

核心问题：为什么二级缓存不行？

如果只是简单的循环依赖（没有AOP），其实二级缓存就够了。把实例化后的对象扔进二级缓存，B拿到A的引用，完成注入，没问题。

但是！如果 A 需要被 AOP 代理呢？

Spring 的设计原则是：Bean 的生命周期中，AOP 代理通常在“初始化后”才创建。

但在循环依赖场景下，B 需要注入 A。如果 A 有切面，B 必须拿到 A 的代理对象，而不是原始对象。

二级缓存存的是原始对象，它不知道要不要变成代理。
三级缓存存的是工厂(lambda)，这个工厂知道如何判断并提前创建代理对象。
