🔀 MVC 类型转换：Converter、Formatter、FormattingConversionService

Converter(S→T) vs Formatter(String↔Object+Locale) — Web 层的类型转换引擎

Part 1Part 2Part 3

Part 1 · 转换体系

接口签名特点
Converter<S,T>T convert(S source)无状态、任意类型互转
Formatter<T>T parse(text, locale) / print(obj, locale)String↔对象、有 Locale
FormattingConversionService统一的转换服务（Spring Boot 默认）

Web 层中 Formatter 优先于 Converter（因为有 Locale 感知）。非 Web 层 Converter 为主。

Part 2 · 触及灵魂

💡 转换链：一步不够时自动组合

如果请求参数是 "123"，要转换成 Long → 先 String→Integer（内置Converter）+ Integer→Long（内置Converter）= 两步转换链。ConversionService 自动组合。

Part 3 · 面试追问（4 组）

Q

Converter 和 PropertyEditor 的根本区别？

PropertyEditor 有状态、线程不安全、只能String↔Object。Converter 无状态、线程安全、任意类型互转。Spring 3.0+ 推荐 Converter。
