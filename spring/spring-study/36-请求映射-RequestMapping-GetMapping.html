Part 1 · 请求映射速查

// 快捷注解（本质都是 @RequestMapping 的特化）
@GetMapping("/users")        = @RequestMapping(method=GET)
@PostMapping                  = @RequestMapping(method=POST)
@PutMapping / @DeleteMapping / @PatchMapping

// @RequestMapping 完整属性
@RequestMapping(
    value = "/users/{id}",
    method = {RequestMethod.GET, RequestMethod.HEAD},
    params = "version=v2",      // 必须带参数 ?version=v2
    headers = "X-API-Key=xxx",  // 必须带 header
    consumes = "application/json", // Content-Type 匹配
    produces = "application/json"  // Accept 匹配
)

Part 2 · 触及灵魂

🤔 为什么需要 consumes/produces？

同一个 URL 可以根据 Content-Type 路由到不同方法。POST /api/users, Content-Type: application/json → createUser()；Content-Type: multipart/form-data → uploadUsers()。不靠 URL 区分，靠请求头区分。

💡 @RequestMapping 的匹配优先级

具体 > 模糊。/users/{id} > /users/* > /users/**。带 consumes/produces/params 的 > 不带限制的。AntPathMatcher 逐条评分，最高分匹配。

Part 3 · 面试追问（6 组）

Q

@GetMapping 和 @RequestMapping(method=GET) 完全等价吗？

源码层面 @GetMapping 就是 @RequestMapping(method=GET)。完全等价。但 @GetMapping 更可读。

Q

两个 @GetMapping("/users") 在一个 Controller 里会怎样？

启动时抛 IllegalStateException——RequestMappingHandlerMapping 不允许同一个 URL+Method 映射到两个方法。

Q

🔧 场景题：API 版本管理 — /v1/users 和 /v2/users 怎么共存？

方案① URL 前缀：/v1/users, /v2/users——最简单。方案② Header：@RequestMapping(headers="API-Version=v2")——URL 不变。方案③ params：@RequestMapping(params="version=v2")。

Q

🔧 场景题：同一个 POST /api/users，怎么根据 Content-Type JSON vs XML 路由到不同方法？

consumes 属性。一个方法 @PostMapping(consumes="application/json")，另一个 consumes="application/xml"。Spring 根据请求 Content-Type 匹配。

Q

AntPathMatcher 和 PathPattern 区别？

AntPathMatcher 基于字符串匹配（古老），PathPattern 基于预编译的路径模式（Spring 5.3+，性能高 50%+）。Spring Boot 2.6+ 默认 PathPattern。

📋 面试官总结

1. 快捷注解是 @RequestMapping 的语法糖

2. consumes/produces/params/headers 精确路由

3. PathPattern 取代 AntPathMatcher
