🚦 前端控制器：DispatcherServlet

前端控制器模式的核心 — 所有 HTTP 请求的"中央车站"

Part 1Part 2Part 3

Part 1 · DispatcherServlet 是什么

DispatcherServlet 是一个Servlet（继承 HttpServlet），它拦截所有进入 Spring MVC 的 HTTP 请求，然后把请求"分发"给合适的 Controller 方法。

// DispatcherServlet.doDispatch() — 核心分发逻辑
HandlerExecutionChain handler = getHandler(request);   // 1. 找到谁处理
HandlerAdapter ha = getHandlerAdapter(handler);         // 2. 找到怎么调用
ModelAndView mv = ha.handle(request, response, handler);// 3. 执行 controller
processDispatchResult(request, response, handler, mv);  // 4. 处理结果

Part 2 · 触及灵魂

🤔 没有 DispatcherServlet 时怎么处理请求？

Servlet 规范中每个 Servlet 映射到不同 URL。10 个功能=10 个 Servlet=10 个 web.xml 配置。路由逻辑、参数解析、视图渲染全部耦合在一个 Servlet 里。DispatcherServlet 用前端控制器模式：一个入口，通过 HandlerMapping 路由到不同 Controller。

💡 设计巧思：三层抽象分离关注点

HandlerMapping（路由）→ HandlerAdapter（调用）→ ViewResolver（渲染）。每层独立替换。可以加自定义 HandlerMapping、自定义 HandlerAdapter、自定义 ViewResolver——DispatcherServlet 不关心具体实现。

⚠️ Spring Boot 如何注册 DispatcherServlet

在 refresh() 的 onRefresh() 步骤中，ServletWebServerApplicationContext 创建内嵌 Tomcat 并注册 DispatcherServlet。不再需要 web.xml——Servlet 3.0+ 的 ServletContainerInitializer 机制让 Spring Boot 可以编程式注册。

Part 3 · 面试追问（6 组）

Q

DispatcherServlet 和普通的 Servlet 是什么关系？

DispatcherServlet 继承 FrameworkServlet → HttpServletBean → HttpServlet。它就是一个 Servlet，只是内部实现了完整的分发逻辑。

A
Q

DispatcherServlet 可以有多实例吗？

可以——每个 DispatcherServlet 有自己的 WebApplicationContext（子容器），共享根容器。适合 REST API 和页面渲染分开的场景。

A
Q

getHandler 返回 null 会怎样？

返回 404。noHandlerFound() 调 response.sendError(404)。如果配置了 throwExceptionIfNoHandlerFound=true，则抛 NoHandlerFoundException。

A
Q

🔧 场景题：线上发现某些请求返回 404 但 Controller 明明存在。怎么排查？

① 开 DEBUG 日志看 RequestMappingHandlerMapping 的映射表 ② 检查 URL 和 Controller 的 @RequestMapping 是否匹配 ③ 检查是否被拦截器 preHandle 返回 false 阻断了 ④ 检查是否有多个 DispatcherServlet，请求进了错的 Servlet。

A
Q

BuilderAdapter 做什么的？

HandlerAdapter 负责统一调用接口——不同 Handler 类型（@RequestMapping Controller / HttpRequestHandler / Servlet）有不同调用方式，HandlerAdapter 把差异屏蔽，给 DispatcherServlet 统一接口。

A
Q

DispatcherServlet 的 init 策略有哪些？

init-param 可配置：detectAllHandlerMappings（查所有 HandlerMapping 还是只取第一个）、throwExceptionIfNoHandlerFound（404 抛异常）、dispatchOptionsRequest（是否处理 OPTIONS 请求）。

A
