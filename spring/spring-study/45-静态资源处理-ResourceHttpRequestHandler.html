Part 1 · Spring Boot 默认静态资源路径

classpath:/static/          → /static/logo.png → GET /logo.png 可访问
classpath:/public/          → /public/index.html
classpath:/resources/       → /resources/
classpath:/META-INF/resources/  → jar 包中自带资源

# 自定义
spring.web.resources.static-locations=classpath:/custom/
spring.mvc.static-path-pattern=/assets/**

Part 2 · 触及灵魂

💡 ResourceHttpRequestHandler 工作原理

它是 Spring MVC 的一个 Handler（不是 Servlet）。SimpleUrlHandlerMapping 把 /assets/** 映射到 ResourceHttpRequestHandler。Handler 内部用 ResourceResolver 链查找资源：PathResourceResolver（路径匹配） → CachingResourceResolver（缓存）→ GzipResourceResolver（压缩）。

Part 3 · 面试追问

Q

静态资源和 Controller 优先级谁高？

Controller 优先。RequestMappingHandlerMapping 的 order 比 SimpleUrlHandlerMapping 小→先匹配。Controller 匹配不了的 URL 才落到静态资源。
