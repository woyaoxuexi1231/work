Part 1 · @Scheduled 三种模式

// fixedRate：两次开始间隔固定（不管上次是否完成）
@Scheduled(fixedRate = 5000)  // 每5秒触发一次

// fixedDelay：上次结束到下次开始间隔固定
@Scheduled(fixedDelay = 5000) // 上次完成后等5秒再执行

// cron：任意时间表达式
@Scheduled(cron = "0 0 2 * * ?") // 每天凌晨2点

// 参数化
@Scheduled(fixedDelayString = "${task.delay:5000}")

Part 2 · 触及灵魂

🤔 fixedRate vs fixedDelay 的核心区别

fixedRate(5000)：不管上次执行多久，第0秒、第5秒、第10秒触发。如果上次耗时6秒→第5秒时会排队（第二个任务等第一个完成）。fixedDelay(5000)：上次执行完后等5秒→如果耗时6秒，实际间隔=6+5=11秒。

💡 @Async 失效场景

同类自调用（this.asyncMethod()）不生效——和 @Transactional 同因。解法：拆类或注入自己。

⚠️ @Scheduled 默认单线程

所有 @Scheduled 默认共享一个线程。一个任务堵了→其他任务全部延迟。必须自定义线程池：@Bean 注册 TaskScheduler 或配置 spring.task.scheduling.pool.size。

Part 3 · 面试追问（7 组）

Q

fixedRate 任务上次没执行完怎么办？

默认：排队等待（任务放入队列）。新任务不会并发执行（除非你开了线程池，但 @Scheduled 默认单线程）。如果不想排队→把耗时任务提交到独立线程池。

Q

🔧 场景题：3 个 @Scheduled 任务，一个阻塞了另外两个也停了。怎么排查和解决？

原因：@Scheduled 默认单线程。解决：配置 spring.task.scheduling.pool.size=4 或 @Bean TaskScheduler（线程数>3）。排查：jstack 看调度线程在哪阻塞。

Q

🔧 场景题：@Async 默认线程池是 SimpleAsyncTaskExecutor——每任务新建线程。高并发下线程数爆炸 OOM。怎么改？

@Bean(name="taskExecutor") ThreadPoolTaskExecutor：core=10, max=50, queue=200。Spring Boot 自动检测到后替换默认的 SimpleAsyncTaskExecutor。或者配置 spring.task.execution.pool.core-size/max-size。
