📊 JdbcTemplate & NamedParameterJdbcTemplate

模板方法模式的经典应用 — 15 行样板代码变成 1 个 lambda

Part 1 · 基本用法

对比原始 JDBCJdbcTemplate
获取连接DriverManager.getConnection()自动从 DataSource
异常处理SQLException(受检)+try-finally自动翻译DataAccessException(非受检)
资源释放手动 close(6行)自动释放
代码量~15行/查询~3行/查询
// JdbcTemplate
@Autowired JdbcTemplate jdbc;
User u = jdbc.queryForObject("SELECT * FROM users WHERE id=?", new BeanPropertyRowMapper<>(User.class), 1L);
List<User> list = jdbc.query("SELECT * FROM users WHERE age>?", rowMapper, 18);
int rows = jdbc.update("UPDATE users SET name=? WHERE id=?", "张三", 1L);

// NamedParameterJdbcTemplate — 具名参数（:name 优于 ?）
@Autowired NamedParameterJdbcTemplate named;
Map<String,Object> p = Map.of("id",1L,"name","张三");
named.update("UPDATE users SET name=:name WHERE id=:id", p);

Part 2 · 触及灵魂

💡 模板方法模式 — JdbcTemplate 的灵魂
// 固定骨架：获取连接→创建Statement→设参→执行→映射→翻译异常→释放
// 变化部分：SQL + 参数 + RowMapper（回调）
// 15 行样板 → 框架内部 / 3 行业务代码 → lambda

"啊哈时刻"：这就是模板方法模式的威力——不变的流程归框架，变化的细节归你。RestTemplate、TransactionTemplate 都是同样的设计。

⚠️ 陷阱

queryForObject 期望恰好 1 条——0 条→EmptyResultDataAccessException，>1 条→IncorrectResultSize。用 query()+Optional 更安全。大数据量不要用 query() 全量加载——用 RowCallbackHandler 流式处理。

Part 3 · 面试追问（6 组）

Q

JdbcTemplate 用什么设计模式？

模板方法模式——固定"获取连接→创建Statement→设参→执行→映射→翻译异常→释放"骨架，只把"映射结果"通过 RowMapper 回调留给开发者。RestTemplate、TransactionTemplate 同理。

A
Q

NamedParameterJdbcTemplate 比 ? 好在哪里？

? 按位置绑定——顺序错了不报错但数据错乱。:name 按名称绑定——顺序无关、更可读。底层还是 JdbcTemplate，只是多了一层参数名→位置的翻译。

A
Q

🔧 场景题：一个查询返回 50 万条数据 OOM 了，怎么改？

① RowCallbackHandler 流式逐条处理 ② 分页查询 ③ ResultSet.TYPE_FORWARD_ONLY+fetchSize 流式读取。JdbcTemplate.query() 默认全量加载到 List——大数据量下是灾难。

A
Q

JdbcTemplate 是连接池吗？每次操作新建连接吗？

不是。JdbcTemplate 每次操作从 DataSource 获取连接，用完释放。连接池是 HikariCP/DBCP 的事——它们管理 DataSource 内部的连接复用。

A
Q

query() 和 queryForList() 的区别？

queryForList 返回 List<Map<String,Object>>（不需要 RowMapper，自动列名→值映射）。query 需要 RowMapper 手动映射。queryForList 快速但类型不安全，query 类型安全但需要多写 RowMapper。

A
Q

JdbcTemplate 内部怎么管理 PreparedStatement？

每次都创建新的 PreparedStatement——不缓存。JDBC 驱动层可能有 statement 缓存（如 MySQL 的 useServerPrepStmts），但那不是 JdbcTemplate 的事。

📋 面试官总结

1. 模板方法模式 — 消灭样板代码的核心设计

2. NamedParameterJdbcTemplate — 具名参数 > 位置参数

3. queryForObject 陷阱 — 0条/多条都抛异常

4. 大数据量用 RowCallbackHandler — 别用 query() 全量加载
