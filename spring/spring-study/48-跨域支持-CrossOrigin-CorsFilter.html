Part 1 · 三种跨域配置

// 方式1：方法级
@CrossOrigin(origins = "https://example.com")
@GetMapping("/users") ...

// 方式2：全局 CorsFilter (推荐)
@Bean CorsFilter corsFilter() { return new CorsFilter(corsConfigSource()); }

// 方式3：WebMvcConfigurer
registry.addMapping("/api/**").allowedOrigins("https://example.com");

Part 2 · 触及灵魂

💡 预检请求 (Preflight) 是什么？

浏览器在跨域 POST/PUT/DELETE 前先发 OPTIONS 请求，问服务器"你允许我吗？"。CorsFilter 自动处理 OPTIONS，不需要写 Controller。如果 OPTIONS 请求进到了 Controller→CORS 配置没生效。

Part 3 · 面试追问

Q

CorsFilter 和 @CrossOrigin 哪个更优先？

CorsFilter 是 Filter 层面的（更外层），@CrossOrigin 是 Handler 层面的（更内层）。CorsFilter 先拦截→处理 OPTIONS→放行→DispatcherServlet→@CrossOrigin 再做一遍。生产建议用 CorsFilter 或 WebMvcConfigurer 全局配置。
