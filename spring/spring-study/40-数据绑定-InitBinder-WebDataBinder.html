Part 1 · @InitBinder 基本用法

// 注册自定义编辑器或设置允许字段
@InitBinder
public void initBinder(WebDataBinder binder) {
    // 1. 注册日期格式化器
    binder.addCustomFormatter(new DateFormatter("yyyy-MM-dd"));
    // 2. 字段白名单（防止 mass assignment 攻击）
    binder.setAllowedFields("name", "email", "age");
    // 3. 字段黑名单
    binder.setDisallowedFields("id", "role");
}

Part 2 · 触及灵魂

💡 Mass Assignment 攻击与字段白名单

如果 User 对象有 isAdmin 字段，攻击者在请求中加 &isAdmin=true→自动绑定到对象→提权。@InitBinder 的 setAllowedFields 是防御之一。更好的方案：用 DTO 而非 Entity 接收参数，DTO 只包含允许修改的字段。

Part 3 · 面试追问

Q

WebDataBinder 的绑定流程？

ServletRequestDataBinder.bind()→遍历请求参数→通过 PropertyAccessor set 到目标对象→PropertyEditor/Formatter 做类型转换。@InitBinder 在 bind 之前执行。

Q

🔧 场景题：前端传 "2025-01-15"，后端要绑定到 LocalDate。怎么统一配置？

@InitBinder 注册 DateFormatter。或全局配置：@ControllerAdvice + @InitBinder。最简单的：spring.mvc.format.date=yyyy-MM-dd。
