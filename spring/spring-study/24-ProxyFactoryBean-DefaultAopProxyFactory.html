🏭 ProxyFactoryBean & DefaultAopProxyFactory

手动创建代理 vs 自动代理 — Spring AOP 的两条创建路径

Part 1Part 2Part 3

Part 1 · 基础认知

两个类的职责分工

类职责角色
ProxyFactoryBean手动创建单个代理对象（FactoryBean）面向使用者
DefaultAopProxyFactory决定用 JDK 还是 CGLIB（createAopProxy方法）代理创建的决策引擎
// ProxyFactoryBean — 手动创建代理（很少直接用）
ProxyFactoryBean factory = new ProxyFactoryBean();
factory.setTarget(new UserServiceImpl());
factory.addAdvice(new LoggingAdvice());
factory.setProxyTargetClass(true);  // 强制 CGLIB
UserService proxy = (UserService) factory.getObject();

// DefaultAopProxyFactory — 内部决策（不直接使用）
public AopProxy createAopProxy(AdvisedSupport config) {
    if (hasNoUserSuppliedProxyInterfaces(config)) {
        return new ObjenesisCglibAopProxy(config);  // CGLIB
    }
    return new JdkDynamicAopProxy(config);  // JDK
}

Part 2 · 触及灵魂

🤔 有自动代理了，为什么还要 ProxyFactoryBean？

自动代理（AbstractAutoProxyCreator）的底层就是 ProxyFactoryBean 的逻辑。ProxyFactoryBean 是手动创建代理的工具——在需要精确控制代理创建过程时使用（比如动态添加 Advisor、运行时决定代理策略）。自动代理是批量化的 ProxyFactoryBean。

💡 DefaultAopProxyFactory 的决策逻辑
// 决策链（伪代码）
if (proxyTargetClass == true)            → CGLIB (ObjenesisCglibAopProxy)
if (目标类没有实现接口)                   → CGLIB
if (目标类实现了接口 && proxyTargetClass未设) → JDK (JdkDynamicAopProxy)

Spring Boot 2.x 把 proxyTargetClass 默认为 true → 几乎永远走 CGLIB。

⚠️ 边界

ProxyFactoryBean 创建的对象是单例（FactoryBean.isSingleton()=true）。如果需要原型代理，设置 singleton=false。

Part 3 · 面试追问（6 组）

Q

ProxyFactoryBean 和 @Aspect 自动代理是什么关系？

自动代理（AnnotationAwareAspectJAutoProxyCreator）的内部实现用的是 ProxyFactory 的逻辑（ProxyFactoryBean 的父类）。一个手动、一个自动；一个单个、一个批量；底层代理创建逻辑完全相同。

A
Q

DefaultAopProxyFactory 在哪里被调用？

在 ProxyFactoryBean.getObject() → ProxyCreatorSupport.createAopProxy() 中被调用。自动代理模式下，AbstractAutoProxyCreator.createProxy() 也会创建 ProxyFactory 并最终调用到它。

A
Q

🔧 场景题：一个第三方 Bean 没有接口，但你需要给它加缓存逻辑（@Cacheable）。这个 Bean 是 final 类，CGLIB 也代理不了。怎么办？

写一个 Wrapper 类（非 final），组合第三方 Bean，在 Wrapper 上加 @Cacheable。Wrapper 暴露和第三方 Bean 一样的接口方法。本质是做了一层代理，但绕过了 CGLIB 的 final 限制。

A
Q

ProxyFactoryBean 的 singleton 属性是什么意思？

决定 getObject() 返回的是不是同一个代理实例。默认 true——每次 getObject() 返回同一个代理。false 时每次返回新代理（目标对象也重新创建）。

A
Q

addAdvice 和 addAdvisor 的区别？

Advice 是纯通知（做什么），Advisor = 通知 + 切入点（做什么 + 在哪里做）。addAdvice 通常搭配默认切点（所有方法），addAdvisor 可以精确控制切点。

A
Q

ObjenesisCglibAopProxy 和 CglibAopProxy 什么区别？

ObjenesisCglibAopProxy 使用Objenesis 绕过构造器创建代理实例——避免执行目标类的构造器逻辑（连接池初始化等副作用）。CglibAopProxy（旧版）会触发构造器。Spring 4+ 默认用 Objenesis 版本。

📋 面试官总结

1. ProxyFactoryBean — 手动创建代理的 FactoryBean，自动代理的底层

2. DefaultAopProxyFactory — JDK vs CGLIB 的决策引擎

3. Objenesis — 绕过构造器，避免 CGLIB 代理触发副作用

4. Advice vs Advisor — 纯通知 vs 通知+切点
