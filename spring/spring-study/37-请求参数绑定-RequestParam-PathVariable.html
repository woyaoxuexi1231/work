Part 2 · 触及灵魂

💡 HandlerMethodArgumentResolver — 参数解析的幕后英雄

Spring 有 26+ 个 ArgumentResolver 实现。每个注解对应一个 Resolver：RequestParamMethodArgumentResolver、PathVariableMethodArgumentResolver、RequestResponseBodyMethodProcessor...。DispatcherServlet 调用 HandlerAdapter 时，遍历 Resolver 链找 supportsParameter()=true 的那一个来解析参数。

💡 @RequestParam 的 required 和 defaultValue

required=true（默认）→ 参数缺失抛 MissingServletRequestParameterException(400)。required=false → null（或 Optional）。defaultValue="0" → 有默认值时 required 自动变 false。Integer/int 的区别：int 不能是 null → 必须带 defaultValue。

Part 3 · 面试追问（6 组）

Q

@RequestParam 不指定 name 时怎么确定参数名？

Java 8+ 加 -parameters 编译时保留参数名→直接用方法参数名。否则用 class 字节码中的 LocalVariableTable。都没有→只能显式指定 name。

Q

🔧 场景题：自定义 @CurrentUser 注入当前登录用户。怎么实现？

实现 HandlerMethodArgumentResolver：supportsParameter 检查 @CurrentUser，resolveArgument 从 SecurityContextHolder 取当前用户。注册到 WebMvcConfigurer.addArgumentResolvers()。Controller 方法只需 public Result profile(@CurrentUser User user)。

Q

@RequestBody 多个参数可以吗？

不可以——HTTP 请求体只能读一次（流式）。多个 @RequestBody 会报错。如果确实需要多个对象→包装成一个 DTO 或一个用 @RequestBody 一个用 @RequestParam。
