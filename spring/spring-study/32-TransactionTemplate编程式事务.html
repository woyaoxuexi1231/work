@Autowired TransactionTemplate tx;
tx.execute(status -> {
    orderMapper.insert(o);
    if (o.getAmount() > 10000) { status.setRollbackOnly(); return null; }
    return null;
});

Part 2 · 触及灵魂

🤔 什么时候必须用编程式事务？

①事务边界小于方法：方法前半段不需要事务，后半段需要。②批量处理：100条记录每条独立事务，失败3条不影响其余97条。③运行时动态决定传播行为（根据配置选REQUIRED或REQUIRES_NEW）。

💡 setRollbackOnly() vs 抛异常

setRollbackOnly() 标记回滚但不抛异常——方法继续执行。抛异常会中断方法。适合"我知道这个操作失败但不希望中断调用链"。

⚠️ TransactionTemplate 和 @Transactional 底层相同

最终都调用 PlatformTransactionManager。TransactionTemplate 只是编程式入口，@Transactional 通过 TransactionInterceptor 间接调用。一台引擎，两种方向盘。

Part 3 · 面试追问（6 组）

Q

TransactionTemplate 和 @Transactional 底层一样吗？

一样——最终都调 PlatformTransactionManager.getTransaction/commit/rollback。一台引擎，两种方向盘。

A
Q

setRollbackOnly() 什么时候用？

需要标记回滚但不想中断方法执行时。比如批量处理第5条失败→标记回滚→continue处理第6条。效果等同抛异常但不中断调用链。

A
Q

🔧 场景题：批量导入1万条数据，每条独立事务。怎么写？

在循环内 TransactionTemplate.execute()——每条一个独立事务。不能用 @Transactional 包整个方法（一个事务包1万条）。失败的记录记error list并continue。

A
Q

TransactionTemplate 能指定传播行为吗？

能。构造时 setPropagationBehavior(REQUIRES_NEW)。或 new TransactionTemplate(tm, new DefaultTransactionDefinition(REQUIRES_NEW))。

A
Q

execute 里 return null 和 return 对象有什么区别？

return 的值就是 execute() 的返回值。不需要返回值时 return null。return 不会触发提交——execute 正常结束自动提交。

A
Q

TransactionTemplate 内部怎么处理异常的？

回调抛 RuntimeException→自动 rollback。Throwable（非RuntimeException）→rollback 后包装成 UndeclaredThrowableException 再抛。你可以自定义 setRollbackOnCommitFailure 行为。

A

📋 面试官总结

1. 同一引擎 — TransactionTemplate 和 @Transactional 底层完全一致

2. 精确控制 — 事务边界小于方法时用编程式

3. 批量独立事务 — 循环内 execute，每条独立提交
