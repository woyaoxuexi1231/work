📋 Q16 · MVC 请求流程 — DispatcherServlet 的完整链路。

面
一个 HTTP 请求从 DispatcherServlet 到最终返回结果，经过了哪些组件？
我
Filter 链 → DispatcherServlet.doDispatch() →
① HandlerMapping：根据 URL 找 Handler，返回 HandlerExecutionChain（Handler + Interceptors）
② HandlerAdapter：适配不同 Handler 类型，反射调用 Controller
③ 参数解析：HandlerMethodArgumentResolver 处理 @RequestParam/@RequestBody/@PathVariable
④ 返回值处理：@ResponseBody → HttpMessageConverter 序列化；String → ViewResolver 渲染

为什么需要 HandlerAdapter？适配器模式。早期 Controller 要实现 Controller 接口，后来引入 @RequestMapping 注解驱动——两种 Handler 调用方式不同。Adapter 让 DispatcherServlet 不需要知道这些差异。
追问："HandlerMapping 怎么找到 Controller 的？"
答：RequestMappingHandlerMapping 启动时扫描所有 @Controller 类的 @RequestMapping 注解，建立 URL→HandlerMethod 的映射表。请求来了直接查表——不是每次请求都扫描。
❓ 自问：DispatcherServlet 的本质是什么？
答：一个策略模式大集合——HandlerMapping、HandlerAdapter、ViewResolver、HandlerExceptionResolver 全部是策略接口。它只负责组装和调度，不负责具体实现。
