📋 Q18 · 拦截器 vs 过滤器 — Token 校验用哪个？

面
HandlerInterceptor 和 Filter 的区别？Token 校验用哪个？
我
Filter 是 Servlet 规范——拦截所有请求，不依赖 Spring，可以注入 Spring Bean 但需要额外配置。Interceptor 是 Spring MVC 的——只拦截进入 DispatcherServlet 的请求，天然支持 @Autowired。

Token 校验推荐 Interceptor。三个原因：可以注入 TokenService；知道是哪个 Controller 方法被调用（可以做 @NoAuth 注解排除）；校验失败可以直接写 JSON。

不是二选一，是分层。Filter 做基础设施（编码、CORS、安全头），Interceptor 做业务逻辑（认证、权限、日志）。执行顺序：Filter → DispatcherServlet → Interceptor.preHandle → Controller。
❓ 自问：什么时候必须用 Filter？
答：需要在 Spring 解析参数之前做的事情——比如 body 加密解密、XSS 过滤、请求日志记录原始 body。这些必须在 Filter 层做，因为到 Interceptor 时 body 已经被 Spring 消费了。
