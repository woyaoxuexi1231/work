📋 Q17 · 参数绑定与转换 — HttpMessageConverter 的设计。

面
@RequestParam、@RequestBody、@PathVariable 的底层实现有何不同？
我
@RequestParam 和 @PathVariable 不用 HttpMessageConverter。它们直接从 request.getParameter() 或 URL 模板提取字符串，然后用 TypeConverter 做 String→Integer 的简单转换。

只有 @RequestBody 走 HttpMessageConverter。因为它要解析 HTTP Body 的字节流——可能是 JSON、XML、Form 数据。Spring 根据 Content-Type 遍历 Converter 链：第一个 canRead() 返回 true 的就用来反序列化。

FormHttpMessageConverter 处理表单（application/x-www-form-urlencoded），MappingJackson2HttpMessageConverter 处理 JSON。新增格式（如 Protobuf）只需加一个 Converter——Controller 代码不用改。
❓ 自问：为什么 URL 参数不用 Converter 链？
答：URL 参数是简单 key-value 字符串——TypeConverter 足够。Body 是结构化数据，需要 Converter 链。不用一种方式处理所有——这是单一职责。
