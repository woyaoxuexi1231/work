MyBatis 完整链路：从 XML/注解 → SQL 执行

四个阶段 · 一个动画 · 一次看明白

1

启动时：XML / 注解 → MappedStatement

MyBatis 在启动阶段把每条 SQL 解析成一个 MappedStatement 对象，存入 Configuration。

📦 MappedStatement 结构
public class MappedStatement { private String id; // = "com.example.UserMapper.findById" private SqlSource sqlSource; // SQL 模板: "SELECT * FROM users WHERE id = ?" private SqlCommandType type; // SELECT / INSERT / UPDATE / DELETE private ResultMap resultMap; // 列 → 字段 映射 // ... }
🗄️ Configuration 内部
protected final Map<String, MappedStatement> mappedStatements = new HashMap<>(); // 启动后，Map 里的内容： // "com.example.UserMapper.findById" → MappedStatement(SELECT * FROM users WHERE id = ?) // "com.example.UserMapper.insert" → MappedStatement(INSERT INTO users ...) // "com.example.OrderMapper.findById" → MappedStatement(SELECT * FROM orders WHERE id = ?)

💡 XML 和注解只是表达方式不同，最终产物都是 MappedStatement。存入 Configuration 的 mappedStatements 这个 HashMap，key = 接口全限定名 + "." + 方法名。

2

Spring 启动时：@MapperScan → FactoryBean → JDK 代理

接口没有实现类——所以 Spring 不直接实例化它，而是用 MapperFactoryBean 这个工厂来生成 JDK 动态代理。

🔍@MapperScan
扫描接口
→
📦BeanDefinition
beanClass=MFB
→
🏭MapperFactoryBean
getObject()
→
🪞$Proxy
JDK 代理对象
🏭 MapperFactoryBean.getObject()
public T getObject() { return (T) Proxy.newProxyInstance( mapperInterface.getClassLoader(), // ClassLoader new Class<?>[]{mapperInterface}, // 要实现哪个接口 new MapperProxy<>(sqlSession, mapperInterface) // ★ InvocationHandler ); }

生成的 $Proxy0 实现了 UserMapper 接口。每个方法的方法体就一行：return h.invoke(this, method, args); — 全部转发给 MapperProxy。

调用时：MapperProxy.invoke() → 查 MappedStatement

代理对象的方法被调用时，MapperProxy 用接口全限定名 + "." + 方法名作为 key，去 Configuration 的 Map 里查 MappedStatement。

🗄️执行 SQL
🎯 MapperProxy.invoke() — 核心逻辑
public Object invoke(Object proxy, Method method, Object[] args) { // ① Object 的方法 (toString/hashCode/equals) 直接执行 if (Object.class.equals(method.getDeclaringClass())) { return method.invoke(this, args); } // ② 从缓存获取或新建 MapperMethod MapperMethod mapperMethod = cachedMapperMethod(method); // ③ ★ 执行——MapperMethod 内部会查 MappedStatement return mapperMethod.execute(sqlSession, args); }
📋 MapperMethod.execute() — 怎么找到对应的 SQL
public Object execute(SqlSession sqlSession, Object[] args) { // ★ 拼 statementId = 接口全限定名 + "." + 方法名 String statementId = mapperInterface.getName() + "." + method.getName(); // → "com.example.mapper.UserMapper.findById" // ★ 从 Configuration 的 Map 中直接查！O(1) 的 HashMap.get() MappedStatement ms = configuration.getMappedStatement(statementId); // 根据 SQL 类型（SELECT/INSERT/UPDATE/DELETE）走不同分支 switch (ms.getSqlCommandType()) { case SELECT: // 处理参数：@Param 注解、Map、POJO → 转成 SQL 参数 Object param = convertArgsToSqlCommandParam(args); return sqlSession.selectOne(statementId, param); case INSERT: // ... } }

💡 关键：没有扫描、没有反射查找、没有循环匹配。就是 configuration.getMappedStatement("com.example.mapper.UserMapper.findById") — 一次 HashMap.get()。

SqlSession 往下：参数绑定 → JDBC → 结果映射

拿到 MappedStatement 后，SqlSession 把参数交给 JDBC，执行 SQL，把 ResultSet 映射回 User 对象。

📝 SqlSource 把 SQL 模板 + 参数 → 可执行的 SQL
// MappedStatement 里的 SqlSource 是 SQL 模板（带占位符） SqlSource sqlSource = ms.getSqlSource(); // 模板内容: "SELECT * FROM users WHERE id = ?" ← 注意这个 ? // 调用时，ParameterHandler 把 ? 替换成实际参数 BoundSql boundSql = sqlSource.getBoundSql(param); // → "SELECT * FROM users WHERE id = ?" // 参数列表: [1L] // ParameterHandler 设置 PreparedStatement: preparedStatement.setLong(1, 1L); // 把 ? 替换成 1L
📦 ResultSetHandler — 把数据库返回的行映射成 User 对象
// JDBC 返回 ResultSet: // | id | name | age | // | 1 | Alice | 25 | ResultSet rs = statement.executeQuery(); // ResultSetHandler 根据 ResultMap 做映射: // 列 "id" → user.setId(rs.getLong("id")) // 列 "name" → user.setName(rs.getString("name")) // 列 "age" → user.setAge(rs.getInt("age")) return user; // → User{id=1, name="Alice", age=25}

✅ 完整链路：一次调用，四层接力

配置层：XMLMapperBuilder / MapperAnnotationBuilder → MappedStatement → 存入 Configuration.mappedStatements（一个 HashMap）

代理层：@MapperScan → 为每个接口注册 MapperFactoryBean → getObject() → Proxy.newProxyInstance() → 生成 $Proxy
           代理的 InvocationHandler = MapperProxy

路由层：调用 userMapper.findById(1L) → $Proxy 转发 → MapperProxy.invoke()
           → 拼 key = "com.example.mapper.UserMapper.findById"
           → configuration.getMappedStatement(key) — 一次 HashMap.get()

执行层：SqlSession → Executor → StatementHandler → ParameterHandler (设置 ?) → JDBC → ResultSetHandler (映射回对象)
