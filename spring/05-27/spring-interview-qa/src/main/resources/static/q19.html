QUESTION 19 · MVC

统一异常处理

@ControllerAdvice + @ExceptionHandler 的原理？ResponseBodyAdvice 能做什么？

面试时你可以这样回答

🥉 青铜

"@ControllerAdvice 全局捕获 Controller 异常，@ExceptionHandler 指定处理哪种异常。异常以 JSON 格式返回。ResponseBodyAdvice 可以在 @ResponseBody 序列化前统一包装返回值。"

🥈 白银

"DispatcherServlet 在 Controller 抛异常时调用 processHandlerException()，遍历 HandlerExceptionResolvers。ExceptionHandlerExceptionResolver 是核心——它扫描所有 @ControllerAdvice Bean，把 @ExceptionHandler 方法建立异常类型→处理方法的映射缓存。匹配成功后反射调用，返回值通过 HttpMessageConverter 序列化。ResponseBodyAdvice 是另一个维度：正常返回时，在 @ResponseBody 序列化前拦截，统一包装成 ApiResponse{code, message, data}——Controller 只返回 User，前端始终收到统一格式。"
注意：@ExceptionHandler 方法的优先级：子类异常 > 父类异常。如果同时匹配，Spring 选最精确的。

🥇 王者

"@ControllerAdvice 是 Spring AOP 思想在 Web 层的完美移植。AOP 解决的是'横切关注点如何在 Bean 层集中管理'的问题——@ControllerAdvice 解决的是'横切关注点如何在 Controller 层集中管理'的问题。ExceptionHandler 对应 AOP 的 AfterThrowing，ResponseBodyAdvice 对应 AOP 的 Around。Controller 本身只处理正常路径——异常路径和返回值包装全部由 Advice 完成。这个对称性不是巧合：Rod Johnson 在实现 Spring MVC 时明显沿用了 AOP 的设计语言。认识到这一点，你就能理解为什么 Spring 的 WebFlux 用了完全不同的实现（WebFilter、WebExceptionHandler）却保持了同样的设计模式——因为模式是跨技术栈的。"
面试官心理："AOP 在 Web 层的移植"、"模式是跨技术栈的"——视角从 MVC 细节上升到设计模式层面。
