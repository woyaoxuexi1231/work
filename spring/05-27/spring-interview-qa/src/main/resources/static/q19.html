📋 Q19 · 统一异常处理 — @ControllerAdvice 原理 + ResponseBodyAdvice。

面
@ControllerAdvice + @ExceptionHandler 的实现原理？
我
DispatcherServlet 在 Controller 抛异常时调用 processHandlerException()，遍历 HandlerExceptionResolvers。ExceptionHandlerExceptionResolver 是核心——它扫描所有 @ControllerAdvice Bean，把 @ExceptionHandler 方法建立异常类型→处理方法的映射。匹配成功后反射调用，返回值通过 HttpMessageConverter 序列化为 JSON 返回。

@RestControllerAdvice = @ControllerAdvice + @ResponseBody——直接返回 JSON 而不是视图。
面
ResponseBodyAdvice 能做什么？
我
在 @ResponseBody 序列化前拦截返回值，统一包装成 ApiResponse{code, message, data}。Controller 只返回 User，前端永远收到统一格式。搭配 @ControllerAdvice 的异常处理，异常路径和正常路径的返回格式完全一致。
❓ 自问：ResponseBodyAdvice 和 AOP 环绕通知有什么区别？
答：ResponseBodyAdvice 专门针对 HTTP 响应体，只在返回值序列化之前执行。AOP 环绕通知更通用，可以拦截任何 Spring Bean 的方法。Web 层的统一包装应该用 ResponseBodyAdvice——语义更明确。
