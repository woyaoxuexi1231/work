📋 Q23 · 启动流程 — SpringApplication.run() 的核心工作。

面
SpringApplication.run() 内部做了哪些核心工作？
我
8 个阶段——

① 创建 SpringApplication → 推断应用类型（SERVLET/REACTIVE/NONE）
② 准备 Environment → 加载配置源
③ 创建 ApplicationContext
④ 准备上下文 → 执行 Initializer
⑤ 加载 BeanDefinition → 解析 @Configuration
⑥ refresh() ← Spring IOC 核心：invokeBeanFactoryPostProcessors（解析 @Configuration）→ registerBeanPostProcessors → onRefresh（创建内嵌 Tomcat）→ finishBeanFactoryInitialization（实例化所有单例 Bean）→ finishRefresh
⑦ 回调 ApplicationRunner / CommandLineRunner
⑧ 发布 StartedEvent / ReadyEvent

整个 run() 是模板方法模式——骨架固定，每个阶段都可扩展。
❓ 自问：CommandLineRunner 和 ApplicationRunner 的区别？
答：ApplicationRunner 参数已解析（getOptionNames/getOptionValues），CommandLineRunner 是原始 String[]。需要解析命令行参数的用前者，简单场景用后者。
