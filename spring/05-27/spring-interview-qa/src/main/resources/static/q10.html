📋 Q10 · AOP 原理 — Advisor/Advice/Pointcut 关系和递归链。

面
Advisor、Advice、Pointcut 的关系？
我
Advisor = Advice + Pointcut。
Pointcut：在哪里做？筛选需要增强的方法（切点表达式）。
Advice：做什么？增强逻辑（前置/后置/环绕/异常通知）。
Advisor：把两者打包在一起。

为什么要分开？因为同一个 Advice 可以用不同的 Pointcut 应用到不同的包——Advice 的代码一行不用改。AspectJExpressionPointcut 只匹配方法签名（类名、方法名、参数类型），在初始化时通过反射判断，不是运行时判断。
面
ReflectiveMethodInvocation 的递归调用链是如何工作的？
我
核心机制是递归而非 for 循环。ReflectiveMethodInvocation 维护一个拦截器列表和一个 index。每次 proceed() 调用取下一个拦截器，把 this 传给拦截器的 invoke()。拦截器内部再调用 mi.proceed()——形成递归。

为什么必须用递归？因为环绕通知需要在 proceed() 前后写逻辑。递归的调用栈天然表达"进入→执行→退出"的三段结构。for 循环只能线性执行，没法表达"执行一半暂停、回来再继续"。
❓ 自问：Spring AOP 遵循什么标准？
答：AOP Alliance——一个独立的 AOP 标准组织，定义了 MethodInterceptor 和 MethodInvocation 接口。Spring 遵守这个标准而非自己发明接口。
