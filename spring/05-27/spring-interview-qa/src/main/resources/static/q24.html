📋 Q24 · 配置优先级 — 外部化配置加载顺序 + 多环境组织。

面
外部化配置的加载顺序？命令行参数、环境变量、application.yml 谁先谁后？
我
从高到低——
1. 命令行参数（--server.port=9090）
2. 环境变量（export SERVER_PORT=9090）
3. 系统属性（-Dserver.port=9090）
4. application-{profile}.yml（外部 config/）
5. application-{profile}.yml（classpath）
6. application.yml（外部）
7. application.yml（classpath）
8. @PropertySource
9. 默认值

核心原则：越靠近部署环境，优先级越高。同一个 jar 从开发到生产不用重新打包——开发用 application-dev.yml 的 8081，上线用命令行 --server.port=80 覆盖。Environment 的 PropertySource 是一个有序列表——遍历时第一个非 null 返回的就是最终值。
面
多环境配置如何组织？
我
application.yml 放公共配置，application-{profile}.yml 放环境差异。激活：spring.profiles.active=prod。分组：spring.profiles.group.prod=prod-db,prod-mq。

bootstrap.yml 是 Spring Cloud Config 的概念，Spring Boot 3.x / Cloud 2022+ 已移除，改用 spring.config.import。
❓ 自问：为什么用列表而不是 Map 存储 PropertySource？
答：Map 不支持同名 key 的多值语义——后者会覆盖前者。列表天然支持"遍历取第一个非 null"的覆盖语义。
