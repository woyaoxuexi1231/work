📋 Q21 · 自动装配原理 — spring.factories + 条件注解。

面
@SpringBootApplication 包含哪三个注解？@EnableAutoConfiguration 如何加载自动配置类？
我
@SpringBootApplication = @SpringBootConfiguration（本质 @Configuration）+ @EnableAutoConfiguration（触发自动装配）+ @ComponentScan（扫描组件）。

自动装配核心流程：@EnableAutoConfiguration → @Import(AutoConfigurationImportSelector) → selectImports() → 从所有 jar 的 spring.factories 读取 ~180 个候选配置类 → 逐个检查 @Conditional → 不满足的跳过 → 剩下的生效。

条件注解：@ConditionalOnClass 通过 classLoader.loadClass() 判断类路径；@ConditionalOnMissingBean 通过 beanFactory 判断容器中是否已有；@ConditionalOnProperty 通过 environment 判断配置值。@ConditionalOnMissingBean 是用户能覆盖自动配置的关键。
追问："--debug 启动有什么用？"
答：打印完整的自动配置报告——Positive matches（哪些配置生效了）、Negative matches（哪些被条件注解跳过了、为什么跳过）。排查"为什么我的 Bean 没被创建"的第一步就是看这个报告。
❓ 自问：Spring Boot 3.x 在自动装配上有什么变化？
答：从 spring.factories 改为 META-INF/spring/...AutoConfiguration.imports——一个文件一个目的，语义更清晰。同时为 GraalVM native image 做准备。
