// AOP 的本质伪代码
public Object invoke(target, args) {
    List<Advice> chain = getAdvices(target);

    // 将目标方法包装成最后一个拦截器
    chain.add(new TargetInvoker(target));

    // 启动链条
    return new Invocation(chain, 0).proceed();
}

设计解析

Invocation 对象是一个有状态的迭代器。 每次调用 proceed()，它就将内部指针 +1，并执行对应的拦截器。 当指针越界时，说明所有前置/后置处理都做完了，此时反射调用真实的目标方法。

Professor's Note:

这就是为什么 Around 通知可以控制目标方法是否执行。如果不调用 proceed()，链条就断了，目标方法永远不会被触发。

public class ReflectiveMethodInvocation implements Invocation {

    private final Object target;
    private final Method method;
    private final Object[] args;
    private final List<MethodInterceptor> interceptors;

    // 游标：当前执行到第几个拦截器
    private int currentIndex = -1;

    public ReflectiveMethodInvocation(Object target, Method method, Object[] args,
                                 List<MethodInterceptor> interceptors) {
        this.target = target;
        this.method = method;
        this.args = args;
        this.interceptors = interceptors;
    }

    @Override
    public Object proceed() throws Throwable {
        // 1. 游标后移
        this.currentIndex++;

        // 2. 判断是否还有拦截器
        if (this.currentIndex < this.interceptors.size()) {
            MethodInterceptor interceptor = this.interceptors.get(this.currentIndex);
            // 3. 执行拦截器，并将自身(this)传入，以便拦截器回调 proceed()
            return interceptor.invoke(this);
        } else {
            // 4. 链条走完，反射调用真实目标方法
            return method.invoke(target, args);
        }
    }
}

代理工厂

这里展示 JDK 动态代理的实现。它负责创建外壳。 关键在于 invoke 方法内部，我们不再直接调用 target，而是创建了一个 Invocation 对象并启动它。 

public class JdkDynamicAopProxy implements InvocationHandler {

    private final AdvisedSupport config; // 保存目标对象、拦截器列表等配置

    public Object getProxy() {
        return Proxy.newProxyInstance(
            getClass().getClassLoader(),
            config.getTargetClass().getInterfaces(),
            this
        );
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        // 获取该方法匹配的拦截器链
        List<MethodInterceptor> chain = config.getInterceptors(method);

        // 如果没有拦截器，直接反射调用（优化性能）
        if (chain.isEmpty()) {
            return method.invoke(config.getTarget(), args);
        }

        // 核心：创建调用链对象并启动
        Invocation invocation = new ReflectiveMethodInvocation(
            config.getTarget(), method, args, chain
        );

        return invocation.proceed();
    }
}

优雅的API

底层再复杂，给用户暴露的必须简单。 通过 Builder 模式或者注解驱动，让使用者无感知地织入逻辑。 

// 1. 定义业务服务
public interface UserService {
    void createUser(String name);
}

// 2. 手动组装 AOP (演示原理)
AdvisedSupport config = new AdvisedSupport();
config.setTarget(new UserServiceImpl());

// 添加一个日志拦截器
config.addInterceptor(new MethodInterceptor() {
    public Object invoke(Invocation inv) throws Throwable {
        System.out.println("[Log] Start...");
        Object ret = inv.proceed(); // 执行业务及后续链条
        System.out.println("[Log] End...");
        return ret;
    }
});

// 3. 获取代理对象并使用
UserService proxy = (UserService) new JdkDynamicAopProxy(config).getProxy();
proxy.createUser("Alice");
