## 一、Spring IOC 容器

1. **Bean 生命周期**  
   `BeanFactory` 和 `ApplicationContext` 在加载 Bean 时的区别？从实例化到销毁，Bean 经历哪些阶段？`BeanPostProcessor` 的 `before` 和 `after` 分别在什么时候触发？如果我想在 Bean 属性设置之前修改某个属性值，应该用哪个扩展点？

2. **循环依赖**  
   Spring 如何解决单例 Bean 的循环依赖？三级缓存分别存什么？为什么必须用三级缓存，只用二级行不行？如果 A 和 B 都是代理对象，循环依赖时 AOP 代理是在哪个环节生成的？构造器注入为什么无法解决循环依赖？

3. **依赖注入**  
   `@Autowired` 和 `@Resource` 的查找逻辑有何不同？`@Qualifier` 和 `@Primary` 的优先级？字段注入有什么致命缺陷？为什么官方推荐构造器注入？你在团队里如何强制推行构造器注入？

4. **FactoryBean**  
   `FactoryBean` 和 `BeanFactory` 是一回事吗？MyBatis 的 Mapper 接口是怎么通过 `FactoryBean` 生成代理的？`getObject()` 返回的对象能否被 AOP 增强？

5. **作用域**  
   singleton、prototype、request、session 的作用域原理？prototype Bean 的生命周期由谁管理？如果 singleton Bean 里注入 prototype Bean，如何保证每次都拿到新实例？（`@Lookup`、`ObjectFactory`、`ApplicationContext.getBean`）

6. **Bean 覆盖与冲突**  
   Spring Boot 中同名 Bean 默认允许覆盖吗？`spring.main.allow-bean-definition-overriding` 参数的作用？你遇到过多模块同名 Bean 冲突怎么排错？

## 二、Spring AOP

7. **代理选择**  
   JDK 动态代理和 CGLIB 各自的实现原理、优缺点、适用场景？Spring Boot 2.x 为什么默认改为 CGLIB？`proxyTargetClass=true` 会强制用 CGLIB 吗，接口存在时呢？

8. **内部调用失效**  
   同一个类中，方法 A 调用方法 B，B 上有 `@Transactional` 或 `@Cacheable`，为什么增强失效？怎么解决？（`AopContext.currentProxy()`、注入自身、重构类结构）

9. **拦截链**  
   多个切面作用于同一方法时，执行顺序如何控制？`@Order` 与 `@Priority` 的区别？环绕增强的 `proceed()` 如果不调用会怎样？

10. **AOP 实现原理**  
    `Advisor`、`Advice`、`Pointcut` 的关系？`AspectJExpressionPointcut` 如何判断一个类是否匹配切点？`ReflectiveMethodInvocation` 的递归调用链是如何工作的？

11. **自研简化版 AOP**  
    如果让你自己实现一个支持前置、后置、环绕通知的 AOP 引擎，核心接口怎么设计？如何在 Bean 初始化时织入代理？

## 三、Spring 事务管理

12. **事务传播行为**

13. **事务失效场景**  
    列举至少 5 种 `@Transactional` 失效的情况：（方法非 public、自调用、异常被 catch 未抛出、rollbackFor 设置错误、数据库引擎不支持事务、多线程调用等）

14. **事务实现原理**  
    `@Transactional` 是如何通过 AOP 织入事务的？`PlatformTransactionManager` 的核心方法？`TransactionSynchronizationManager` 的作用？如何在一个事务提交后执行自定义逻辑？（`TransactionSynchronization` 的 `afterCommit`）

15. **分布式事务**  
    只用 Spring 框架，不引入 Seata，你能怎么实现简单的分布式事务？（JTA、TCC 手动实现、本地消息表 + 定时任务 + 幂等）
