### 1. AES —— 就是一个“多次搅拌的替换表”
**逻辑：把数据切成块，照着密码本（S盒）换字，再换位置、搅匀，反复10多次，神仙也认不出原样。**

自定义一个2字节的微型AES：
- **明文块**：`[0x32, 0x88]`
- **密钥**：`[0x2B, 0x28]`
- **S盒**（简单写几个）：`0x32→0x23`, `0x88→0x4A`...
- **行移位**：交换两个字节的位置
- **列混淆**：简单版就定义为 `a = a XOR b; b = a XOR b`（交换并异或）
- **轮密钥加**：和轮密钥异或

**加密一轮走一遍：**
1. **S盒替换**：`[0x32, 0x88]` → 查表得 `[0x23, 0x4A]`
2. **行移位**：交换位置 → `[0x4A, 0x23]`
3. **列混淆**：`0x4A XOR 0x23 = 0x69`，新`a=0x69`，`b = 0x69 XOR 0x23 = 0x4A` → `[0x69, 0x4A]`
4. **轮密钥加**：异或当前轮密钥（假设由密钥扩展得到 `0xA0, 0xB1`） → `0x69 XOR 0xA0 = 0xC9`, `0x4A XOR 0xB1 = 0xFB` → 密文 `[0xC9, 0xFB]`

**解密**就是倒着来：逆轮密钥加 → 逆列混淆 → 逆行移位 → 逆S盒，立刻回到 `[0x32, 0x88]`。  
**真正的AES只是把块变128位，这套搅拌动作重复10-14轮，轮密钥每一轮都不同。**

### 2. RSA —— 你已经门儿清，我换个更小的数字复现
**逻辑：私钥是“几次方取余”，公钥是“另一次方取余”，两个互为逆操作。**

选质数 `p=3, q=11` → `n=33`，欧拉函数 `φ=20`。  
选 `e=3`，算出 `d=7`（因为 `3×7 mod 20 =1`）。
- **公钥**：`(3, 33)`
- **私钥**：`(7, 33)`

**加密**：发 `“M=4”` → 密文 `C = 4³ mod 33 = 64 mod 33 = 31`  
**解密**：`M = 31⁷ mod 33`，计算：  
`31²=961 mod 33=4` → `31⁴=16` → `31⁷=16×4×31=1984 mod 33 = 4`  
直接回4。**核心就是：模幂运算正逆不同，且大数分解n=p×q极难。**

### 3. SHA-256 —— 数据扔进绞肉机，出来固定长度的“指纹”
**逻辑：把消息切成块，和初始魔数一起，经历移位、异或、与或非等几十轮搅打，最后流出一杯固定浓度的“杂凑汁”。**

手工造一个4变量的微型SHA：
- 初始哈希 `A=0x12, B=0x34, C=0x56, D=0x78`
- 消息一个字 `W = 0x41`（字母'A'）
- 每轮压缩函数简化为：  
  `T = (A循环左移3) XOR (B AND C) XOR W`  
  然后 `A=D, D=C, C=B, B=A+T`

**压一轮：**
1. A左移3：`0x12` 是 `0001 0010`，左移3变 `1001 0000` = `0x90`
2. `B AND C`：`0x34 & 0x56 = 0x14`
3. `0x90 XOR 0x14 XOR 0x41` = `0x90 XOR 0x14 = 0x84`，再 XOR `0x41 = 0xC5` → T=0xC5
4. 状态更新：`A=旧D=0x78, D=旧C=0x56, C=旧B=0x34, B=旧A+T=0x12+0xC5=0xD7`  
   新哈希状态：`A=0x78, B=0xD7, C=0x34, D=0x56`

如果把消息从`0x41`改成`0x42`，T立刻面目全非，最终四个值彻底不同。**真正SHA-256有8个初始魔数，消息块扩展到64个字，每轮搅打64次，雪崩效应惊人。**

### 4. 国密 SM4 —— 加密解密用完全一样的算法，只是轮密钥倒着用
**逻辑：Feistel结构，一边保持不变，另一边和轮密钥搅拌后交换，反复32轮。解密时把密钥顺序反过来就行。**

假设明文分成4个4位字：`X0=0x1, X1=0x2, X2=0x3, X3=0x4`  
轮密钥：`rk0=0xA, rk1=0xB, rk2=0xC...`（简化成共3轮）  
每轮运算：`X_{i+4} = X_i XOR T( X_{i+1} XOR X_{i+2} XOR X_{i+3} XOR rk_i )`  
T变换简化为：输入`v`，`T(v) = (v+1) mod 16`（简单的非线性）

**三轮加密：**
- 第1轮：`v = X1 XOR X2 XOR X3 XOR rk0 = 2^3^4^0xA = 0x2^0x3=1, 1^0x4=5, 5^0xA=0xF`  
  T(0xF) = (15+1) mod 16 = 0  
  `X4 = X0 XOR 0 = 0x1`  
  新状态：`X1=2, X2=3, X3=4, X4=1`（丢弃X0）
- 第2轮：`v = X2^X3^X4^rk1 = 3^4^1^0xB = 0xD`，T=14=0xE，`X5=X1 XOR 0xE = 0x2^0xE=0xC`
  状态：`X2=3, X3=4, X4=1, X5=0xC`
- 第3轮：`v = 4^1^0xC^0xC=5`，T=6，`X6= X2 XOR 6 = 3^6=5`
  最终密文按序输出：`X6=5, X3=4, X4=1, X5=0xC`（注意顺序规则）

**解密**时，用**相同的结构，只是轮密钥顺序反过来**（先rk2, 再rk1, 再rk0），即可从 `5,4,1,0xC` 变回 `1,2,3,4`。  
**这就是SM4的核心：加密=解密，全看密钥顺逆。** 真实SM4是128位块，32轮，S盒和线性变换很精致。
