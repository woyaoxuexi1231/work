直观理解 CountDownLatch (一票否决/不可重置) 与 CyclicBarrier (循环屏障/可重置) 的核心区别

CountDownLatch

场景：主线程等待多个子任务完成 (一次性)

剩余计数 (Count)
3
MAIN THREAD AWAIT
Status: Waiting for tasks...
特点： 计数器减至 0 后无法重置。所有任务完成后，主线程才能继续执行（通过终点线）。

CyclicBarrier

场景：多线程互相等待至屏障点 (可循环)

当前轮次 (Cycle)
1
等待中: 0/3
BARRIER ACTION
T1
T2
T3
Status: Ready to run
特点： 所有线程到达屏障点后阻塞，触发 BarrierAction，然后计数器重置，进入下一轮循环。

CountDownLatch (减计数)

  • await(): 阻塞直到计数器为0
  • countDown(): 计数器减1
  • 重用性: ❌ 不可重用 (一次性)
  • 核心: AQS (Shared Mode)

CyclicBarrier (加计数)

  • await(): 阻塞直到所有线程都调用await
  • Action: 人齐后触发公共任务
  • 重用性: ✅ 可重用 (Cyclic)
  • 核心: ReentrantLock + Condition
