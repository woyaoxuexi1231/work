许可证模型

LockSupport 使用二元信号量（Permit）。
unpark() = 发放许可 (最多累积1个)
park() = 消费许可并继续运行，若无许可则阻塞。

顺序无关性

与 wait/notify 不同，LockSupport 不要求先 park 再 unpark。
提前调用 unpark 会预存许可，后续 park 会立即消耗许可而不阻塞。

精准唤醒

unpark(Thread) 直接指定目标线程，无需像 notifyAll 那样唤醒所有等待者再竞争，避免了“虚假唤醒”和惊群效应。
