## 在极端崩溃恢复场景下，InnoDB 相比 MyISAM 具有哪些“崩溃恢复”优势？其底层依赖什么日志？



innodb底层存在 undo日志 和 redo日志，undo日志负责保存原始数据，redo日志负责保存所有的更新操作。

数据崩溃时，查看redo日志检查事务是否提交，如果未提交通过undo日志恢复之前的数据，如果已经提交，但是数据还未保存到磁盘，则通过redo日志进行恢复操作。
