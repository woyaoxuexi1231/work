以下为 **Apache Dubbo**（核心 RPC 框架，版本 2.7/3.x 通用）的完整知识点列表，仅聚焦 Dubbo 本身，不涉及 Dubbo Admin、Dubbo Monitor 等周边系统，也不涉及 Spring Cloud 等其他生态。

---

### Dubbo 知识点

**1. 核心概念与架构**
- 服务提供者（Provider）
- 服务消费者（Consumer）
- 注册中心（Registry）
- 监控中心（Monitor）
- 调用关系与运行容器（Container）
- RPC 调用流程：服务引用、代理、路由、负载均衡、协议编码、网络传输、反序列化、服务端处理
- 地址发现与服务路由
- Dubbo 核心设计原则：微内核 + 插件化（SPI）

**2. 配置方式**
- XML 配置：`<dubbo:service/>`、`<dubbo:reference/>`、`<dubbo:protocol/>` 等
- 注解配置：`@DubboService`（原 `@Service`）、`@DubboReference`（原 `@Reference`）、`@EnableDubbo`
- Java API 配置：`ServiceConfig`、`ReferenceConfig`、`ProtocolConfig` 等
- 属性文件配置：`dubbo.properties`
- 自动配置：`dubbo-spring-boot-starter`，自动识别 `application.properties` 中的 `dubbo.*`
- 配置优先级：JVM 参数 > XML > 属性文件 > 注解/API 默认值
- 多注册中心、多协议配置
- 方法级配置覆盖

**3. 服务注册与发现**
- 注册中心支持：Zookeeper、Nacos、Consul、Etcd、Redis、Sofa、Kubernetes API Server 等
- 注册中心职责：服务注册、服务订阅、变更推送
- 服务分组（group）与版本（version）
- 接口级服务发现（传统）与应用级服务发现（Dubbo 3，元数据与服务映射）
- 注册信息：服务接口、方法、参数类型、URL 属性
- 服务健康检测：注册中心心跳与重连机制
- 临时节点与持久化节点
- 路由信息暂存与缓存

**4. 协议与序列化**
- Dubbo 协议：`dubbo://` 默认协议，TCP 长连接，NIO 异步通讯，Hessian 序列化
- 其他内置协议：`tri://`（Triple，基于 HTTP/2、流式支持）、`http://`、`hessian://`、`rest://`、`thrift://`、`redis://`、`gRPC` 相关等
- 协议配置：端口、线程池、payload 限制、序列化方式
- 序列化扩展：Hessian2、JSON、Protobuf、Kryo、FST、Java、Compact Java
- 序列化性能与安全性
- Triple 协议特性：兼容 gRPC、流式 RPC、HTTP/2、跨语言支持
- 多协议暴露同一服务

**5. 集群容错与负载均衡**
- 集群容错策略：
    - `failfast`：快速失败
    - `failsafe`：安全失败，吞掉异常
    - `failback`：失败自动恢复，后台记录失败请求并定时重发
    - `forking`：并行调用多个服务器，一个成功即返回
    - `broadcast`：广播调用所有提供者，任一失败则报错
    - `available`：遍历可用服务实例
    - `mergeable`：合并结果
- 负载均衡策略：
    - `random`：随机，带权重
    - `roundrobin`：轮询，带权重
    - `leastactive`：最少活跃调用数
    - `consistenthash`：一致性哈希，相同参数请求发同一提供者
- 自定义负载均衡
- 服务端限流与熔断（早期 Sentinel 集成，但 Dubbo 内有简单机制）
- 粘滞连接（`sticky`）
- 结果合并

**6. 远程调用细节**
- 同步调用、异步调用（`CompletableFuture`，事件回调）
- 直连调用（绕过注册中心）
- 多参数调用与参数传递
- 隐式参数传递（`RpcContext` 中的 `attachments`）
- 服务调用超时与重试
- 调用拦截器：`Filter` 链
- 本地存根（`stub`）与服务降级（`mock`）
- 泛化调用：`GenericService`，无需接口类，适用于网关、测试等
- 上下文信息：获取调用方 IP、应用名、令牌验证
- 异步调用三种方式：Future、Callback、事件

**7. SPI 扩展机制**
- Dubbo 自定义 SPI（基于 `@SPI`）
- 扩展点自动加载：`META-INF/dubbo/internal/`、`META-INF/dubbo/`、`META-INF/services/`
- `ExtensionLoader`：获取自适应扩展、激活扩展、指定实现
- 自适应扩展（`@Adaptive`）：动态生成适配类
- 激活扩展（`@Activate`）：按条件激活 Filter 等
- 包装器扩展：AOP 增强
- 扩展点实例化与依赖注入

**8. 过滤器与拦截器**
- Filter 链：请求处理器前/后执行
- 常见内置 Filter：`ActiveLimitFilter`、`ExecuteLimitFilter`、`CacheFilter`、`TokenFilter`、`AccessLogFilter`、`ExceptionFilter`、`TimeoutFilter`、`MonitorFilter`、`TpsLimitFilter` 等
- 自定义 Filter：实现 `Filter` 接口，`@Activate` 分组排序
- 消费端与服务端过滤器

**9. 高级特性**
- 参数验证：Bean Validation 支持（`@DubboService(validation = "true")`）
- 缓存：基于 SPI 的 `Cache` 接口，`lru`、`threadlocal`、`jcache`
- 异步结果缓存与结果转换
- 本地伪装（Mock）：服务降级，返回固定值或异常
- 服务分组与合并结果
- 延迟连接：`lazy="true"`，在真正调用时才建立连接
- 并发控制：`executes` 限制服务端线程数，`actives` 限制消费者并发数
- 连接控制：`connections` 建立连接数，`accepts` 服务端最大连接
- 令牌验证：通过 `token` 防止未授权调用
- 优雅停机：使用 ShutdownHook，标记只读，处理完已接收请求再关闭
- 多版本、多分组服务控制
- 多协议支持与协议前缀

**10. 线程模型与网络层**
- 服务端线程池模型：`fixed`、`cached`、`limited`、`eager`
- 派发策略：`all`（所有消息到线程池）、`direct`（IO 线程处理）、`message`、`execution`、`connection`
- 网络通信框架：Netty（默认）、Mina、Grizzly
- Reactor 线程模型与事件循环

**11. 元数据中心与配置中心（Dubbo 3）**
- 元数据中心：存储服务元数据、方法参数、接口定义
- 应用级服务发现：接口-应用映射关系
- 配置中心：动态配置推送，规则（路由规则、动态配置）热生效
- 服务自省：Provider 端元数据上报

**12. 生态与工具（仅限框架内支持）**
- Telnet 命令：`ls`、`invoke`、`status`、`ps`、`cd` 等
- QoS（Quality of Service）安全管控：在线运维命令、访问控制
- 服务引用与导出事件监听
- 状态检查与健康端点（可集成 Actuator）
- 跨语言支持：Triple 协议下的 gRPC 互通、Rust/Go 等 SDK

**13. 性能与调优**
- 序列化优化选择
- 合理配置超时与重试
- 连接数与线程池调优
- 结果缓存与索引
- 通道复用与连接池
- 关闭自动注册与简化启动

---

以上列表完整覆盖了 Apache Dubbo 核心 RPC 框架自身的所有主要知识点，未引入 Dubbo 管理体系之外的组件或其他 Spring Cloud 生态。
