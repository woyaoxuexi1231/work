# gc-20260609-cpu100 操作指南

用于复现与练习「Java 进程 CPU 突然飙升到 100%」的经典排查链路：

1. top 定位高 CPU 进程
2. top -Hp 定位高 CPU 线程（tid）
3. jstack 抓栈并用 nid=0x... 对齐到具体线程栈
4. 识别死循环/热点方法、BLOCKED、死锁

---

```bash
mvn spring-boot:run
```

mvn -DskipTests package

## 2. 接口说明

基础路径：`http://localhost:8300`

### 2.1 获取 PID（推荐第一步）

```bash
curl http://localhost:8300/lab/pid
```

返回示例：

```json
{"pid":"12345"}
```

### 2.2 制造 CPU 100%（忙等）

```bash
curl -X POST "http://localhost:8300/lab/cpu/spin?threads=4&durationSeconds=300&group=default"
```

参数说明：

- threads：启动多少个忙等线程
- durationSeconds：持续多少秒（到期会自动结束）
- group：分组名，便于启动/停止不同组

停止指定组：

```bash
curl -X POST "http://localhost:8300/lab/cpu/stop?group=default"
```

### 2.3 制造线程死锁

```bash
curl -X POST "http://localhost:8300/lab/deadlock/start"
```

### 2.4 制造 BLOCKED（争抢同一把锁）

```bash
curl -X POST "http://localhost:8300/lab/blocked/start?holdSeconds=600"
```

### 2.5 查看当前状态

```bash
curl http://localhost:8300/lab/status
```

## 3. 排查演练（Linux 经典链路）

本节假设你在 Linux 服务器上操作（线上排查同理）。

### 3.1 top：定位是哪个进程 CPU 高

```bash
top
```

重点看：

- load average：是否持续升高（CPU 跑满时通常会高）
- 进程列表：哪个进程 `%CPU` 异常（找到 Java 进程 pid）

### 3.2 top -Hp <pid>：定位是哪个线程 CPU 高

```bash
top -Hp <pid>
```

重点看：

- 列表中的 tid（线程 id，十进制）
- 哪个 tid 的 `%CPU` 持续最高

记录下最高的 tid，下面要用。

### 3.3 tid 转 nid（十进制转十六进制）

在 Linux 上把 tid 转成十六进制（nid 用的是十六进制）：

```bash
printf "%x\n" <tid>
```

假设输出为 `1a2b`，那么 jstack 里会出现 `nid=0x1a2b`。

### 3.4 jstack：抓取线程栈并对齐到高 CPU 线程

抓栈：

```bash
jstack -l <pid> > jstack.txt
```

查找对应线程：

```bash
grep -n "nid=0x1a2b" -n jstack.txt
```

看到对应线程块后，重点看：

- 线程状态：RUNNABLE / BLOCKED / WAITING / TIMED_WAITING
- 栈顶方法：是否卡在某段循环、计算热点、锁竞争

## 4. 如何通过 jstack 识别常见问题

### 4.1 死循环 / 热点方法（CPU 100% 常见）

特征：

- top -Hp 中某个线程 `%CPU` 持续很高
- jstack 中该线程通常是 `RUNNABLE`
- 栈顶反复停在同一个方法或循环逻辑中

本项目触发方式：

- 调用 `/lab/cpu/spin`
- 高 CPU 线程名类似：`cpu-spin-<group>-<n>`

### 4.2 线程死锁

特征：

- `jstack -l` 输出底部会直接给结论：
  - `Found one Java-level deadlock:`
  - 并列出互相等待的线程与锁

本项目触发方式：

- 调用 `/lab/deadlock/start`
- 线程名类似：`deadlock-1`、`deadlock-2`

### 4.3 BLOCKED（锁竞争）

特征：

- jstack 中很多线程是 `BLOCKED (on object monitor)`
- 同时会看到某个线程持有锁（locked），其他线程在 waiting to lock

本项目触发方式：

- 调用 `/lab/blocked/start`
- 线程名类似：`monitor-holder`、`monitor-blocker`

## 5. 常用补充命令（可选）

### 5.1 多抓几次 jstack 对比（确认“持续卡住”而非瞬时）

```bash
for i in {1..5}; do date; jstack -l <pid> | head -n 80; sleep 2; done
```

### 5.2 只看 Java 进程（快速拿到 pid）

```bash
jps -l
```

## 6. 恢复现场

- CPU 忙等：
  - 等待 durationSeconds 到期自动结束
  - 或调用 `/lab/cpu/stop`
- 死锁：
  - 只能重启进程（或在实验环境直接停止进程）
- BLOCKED：
  - 等 holder 释放锁（holdSeconds 到期）或重启进程
