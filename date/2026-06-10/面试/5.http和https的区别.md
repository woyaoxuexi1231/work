HTTP 和 HTTPS 的核心区别在于**安全**。HTTPS 就是在 HTTP 的基础上，加入了 SSL/TLS 加密层，解决了 HTTP 明文传输、易被窃听和篡改的问题。

具体对比看这里：

| 对比维度 | **HTTP** | **HTTPS** |
| :--- | :--- | :--- |
| **安全性** | 明文传输，数据裸奔，容易被中间人攻击、窃听、篡改。 | 加密传输，能有效防止窃听和篡改。 |
| **端口** | 默认使用 **80** 端口 | 默认使用 **443** 端口 |
| **URL 前缀** | `http://` | `https://`，浏览器地址栏会显示**安全锁**图标。 |
| **加密机制** | 无加密 | 通过 **SSL/TLS** 协议进行加密，包含非对称加密和对称加密的混合机制。 |
| **证书要求** | 不需要证书 | **必须**从受信任的CA（证书颁发机构）申请SSL证书，用于验证服务器身份。 |
| **性能** | 无需加解密和握手协商，速度略快。 | 需要 TLS 握手，早年有性能损耗；但现代优化（如 HTTP/2, TLS 1.3）已让速度差距微乎其微，很多场景下 HTTPS 反而更快。 |
| **SEO 影响** | 搜索引擎对 HTTP 网站的权重较低。 | Google 明确将 HTTPS 作为**排名加分项**，更有利于 SEO。 |

**一句话总结工作原理的不同：**
- **HTTP**：你写一张明信片寄出去，邮递员和所有人都能看到内容。
- **HTTPS**：你把信放进一个密码箱，只有收信人有钥匙打开。即使箱子被截获，没有钥匙也看不到内容，而且箱子上的防拆封条（完整性校验）能证明它是否被打开过。

现在不仅是电商、网银等敏感网站，**几乎所有网站都建议启用 HTTPS**。它已成为现代 Web 的标配，不仅能保护用户数据，还能增加信任感。免费证书（如 Let's Encrypt）的普及也让部署成本变得极低。
