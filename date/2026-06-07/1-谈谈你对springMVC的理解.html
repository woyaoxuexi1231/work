核心定位

Spring MVC 是基于 Servlet API 构建的轻量级 Web 框架。它本质上是 MVC 设计模式在 Java Web 领域的标准化实现，通过高度抽象屏蔽了原生 Servlet 的复杂性，同时利用 Spring IoC/AOP 实现了组件的极致解耦。

核心组件与设计哲学

策略模式 + 模板方法

DispatcherServlet 初始化时加载9大组件。每个组件都是一个策略接口，开发者可以随意替换默认实现（如自定义 HandlerMapping），完美遵循开闭原则。

onRefresh() -> initStrategies()

HandlerAdapter

解决 Controller 实现方式多样性问题。无论是注解式还是接口式，适配器都能统一调用规范并处理参数绑定。

Interceptor

区别于 Servlet Filter，它能访问 Spring 上下文，精细控制到具体 Method 级别，适合登录校验、日志埋点。

MessageConverter

前后端分离的核心。自动将对象序列化为 JSON/XML，替代了传统的 ViewResolver。

ExceptionResolver

配合 @ControllerAdvice 实现全局统一异常处理，解耦业务代码与错误响应逻辑。

DataBinder 体系

强大的类型转换引擎。支持从 String 到复杂对象的自动映射，配合 Validator 接口实现声明式参数校验。
