Thread Ordering Problem

如何确保 T1 → T2 → T3 顺序执行？

面试官考察的不是“怎么创建线程”，而是你对线程间通信机制的理解深度。 从原生 API 到 JUC 工具类，共有 4 种标准解法。

01

join()

最直观 / 串行化

02

CountDownLatch

JUC 首选 / 一次性

03

CompletableFuture

现代异步 / 链式编排

04

LockSupport

底层原语 / 精准控制

Basic

方案一：Thread.join() 串行等待

在主线程中依次启动并 join，本质是将并行退化为串行。简单但失去了多线程的意义。

推荐指数: ★★☆☆☆
Main
start+join
T1
run()
T2
run()
T3
Thread t1 = new Thread(() -> System.out.println("T1"));
Thread t2 = new Thread(() -> System.out.println("T2"));
Thread t3 = new Thread(() -> System.out.println("T3"));

t1.start();
t1.join();  // 主线程阻塞，等T1结束

t2.start();
t2.join();  // 等T1结束后才启动T2

t3.start();
t3.join();
Recommended

方案二：CountDownLatch 门闩机制

每个线程持有下一个线程的门闩。T2 等待 T1 的 latch，T3 等待 T2 的 latch。面试标准答案。

推荐指数: ★★★★★ 
// 为每个线程创建独立的门闩，初始值为1
CountDownLatch latch1 = new CountDownLatch(1);
CountDownLatch latch2 = new CountDownLatch(1);

Thread t1 = new Thread(() -> {
    System.out.println("T1 Running");
    latch1.countDown(); // 释放T2的等待
});

Thread t2 = new Thread(() -> {
    try { latch1.await(); } catch (InterruptedException e) {}
    System.out.println("T2 Running");
    latch2.countDown(); // 释放T3的等待
});

Thread t3 = new Thread(() -> {
    try { latch2.await(); } catch (InterruptedException e) {}
    System.out.println("T3 Running");
});

// 注意：启动顺序可以任意，因为await保证了执行顺序
t3.start();
t1.start();
t2.start();
Modern Java

方案三：CompletableFuture 链式编排

Java 8+ 函数式编程风格。代码最优雅，语义最清晰，适合复杂异步流水线。

推荐指数: ★★★★☆ 
CompletableFuture.runAsync(() -> System.out.println("T1"))
    .thenRunAsync(() -> System.out.println("T2"))
    .thenRunAsync(() -> System.out.println("T3"))
    .join(); // 可选：等待整个链路完成
Advanced

方案四：LockSupport 精准唤醒

AQS 的底层基石。park/unpark 不依赖对象锁，且 unpark 可以先于 park 调用，避免死锁风险。

推荐指数: ★★★☆☆ 
Thread t1, t2, t3;

t1 = new Thread(() -> {
    System.out.println("T1");
    LockSupport.unpark(t2); // 精准唤醒T2
});

t2 = new Thread(() -> {
    LockSupport.park();     // 阻塞自己
    System.out.println("T2");
    LockSupport.unpark(t3); // 精准唤醒T3
});

t3 = new Thread(() -> {
    LockSupport.park();
    System.out.println("T3");
});

// 必须先声明引用，启动顺序任意
t1.start(); t2.start(); t3.start();

🧭 面试选型决策指南

  • 简单演示 / 脚本任务 直接用 join()，代码最少，无需额外依赖。
  • 生产环境 / 并发协调 首选 CountDownLatch，语义明确，JUC 标准组件。
  • 异步流水线 / 结果聚合 用 CompletableFuture，支持异常处理和返回值传递。
  • 框架开发 / 极致性能 用 LockSupport，它是 ReentrantLock 和 CountDownLatch 的底层实现。

避坑提示：不要尝试用 sleep() 或 volatile 标志位来保证顺序！sleep 时间不可靠，volatile 只保证可见性不保证原子性和有序性，在 CPU 重排序下依然会乱序。
