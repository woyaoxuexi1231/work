函数式接口

本质：“仅含一个抽象方法的接口”，Lambda表达式得以依附的类型系统。

✅ 核心定义与注解

  • 使用 @FunctionalInterface 强制约束（编译期检查）
  • 可以包含 default / static 方法，但不影响函数式接口的“函数描述符”
  • 允许重写 Object 类中的 public 方法（如 toString/equals）而不破坏抽象方法计数
  • 典型示例：Runnable, Comparator, Callable

自定义函数式接口 & 传统实现

@FunctionalInterface
interface Calculator {
    int compute(int a, int b);
    // 默认方法不影响函数式接口定义
    default void print() { System.out.println("计算中..."); }
}

// 使用Lambda实现函数式接口实例
Calculator add = (a, b) -> a + b;
Calculator multiply = (a, b) -> a * b;
System.out.println(add.compute(5,3)); // 输出 8
⚡

Lambda 表达式

匿名函数 — 简洁、轻量，将行为参数化的优雅手段。

语法形式： (parameters) -> { body; }
✔ 单行可省略 {} 与 return  ✔ 参数类型可推断  ✔ 只有单个参数时可省略 ()

🔍 典型Lambda示例 & 类型推断

// 1. 无参Lambda (Runnable)
Runnable task = () -> System.out.println("异步执行");

// 2. 带参数单行表达式
List<String> list = Arrays.asList("lambda","java");
list.sort((a, b) -> a.compareTo(b));

// 3. 多行语句块, 需要显式return
Calculator divide = (x, y) -> {
    if (y == 0) throw new IllegalArgumentException("除数不能为0");
    return x / y;
};

// 4. 变量捕获(effectively final)
int factor = 10;
Calculator times = (a,b) -> a * b * factor; // factor 实质final

JDK 四大金刚 · 常用函数式接口

Java 8 在 java.util.function 中提供了丰富预定义接口，彻底拥抱函数式编程。

接口类型描述符抽象方法使用场景
PredicateT -> booleantest(T t)条件过滤、校验
FunctionT -> Rapply(T t)类型转换/映射
ConsumerT -> voidaccept(T t)遍历消费、打印
Supplier() -> Tget()工厂、懒加载数据
UnaryOperatorT -> Tapply(T t)一元运算 (Function子类)

✨ 实战片段: Predicate + Consumer 结合Lambda

List<String> names = Arrays.asList("Java", "Scala", "Kotlin", "Python");
// Predicate 过滤 + Consumer 消费
names.stream()
    .filter(s -> s.startsWith("J") || s.startsWith("K"))
    .forEach(System.out::println); // 方法引用 (更简洁)

// Function 映射长度
Function<String, Integer> lenFunc = str -> str.length();
names.stream().map(lenFunc).forEach(l -> System.out.print(l + " "));

方法引用 (Method Reference)

Lambda的简化糖衣，当Lambda表达式仅调用一个已存在方法时更加优雅。

类静态方法引用
ClassName::staticMethod
例: Math::max, Integer::parseInt
实例方法引用 (特定对象)
instance::methodName
例: list::add, System.out::println
特定类型的任意对象方法
ClassName::instanceMethod
例: String::length, Person::getName
构造方法引用
ClassName::new
例: ArrayList::new, Employee::new
// 等价形式：Lambda vs 方法引用
Consumer<String> printer = s -> System.out.println(s);
Consumer<String> printerRef = System.out::println; // 方法引用更简洁

Function<String, Integer> toLength = str -> str.length();
Function<String, Integer> toLengthRef = String::length;
⚖️

Lambda vs 匿名内部类

this指向Lambda中this指向外部类实例；匿名内部类指向自身
字节码生成Lambda使用invokedynamic，运行时动态生成，避免产生额外类文件；匿名内部类会生成$1.class
变量捕获Lambda捕获effectively final变量；匿名内部类需显式final(Java8前)
性能Lambda通常更轻量，延迟加载且可复用同一函数式接口实例；匿名类每次创建都会加载类
适用范围只能用于函数式接口；匿名内部类可用于抽象类、普通接口多方法
// this 区别示例
public class ThisDemo {
    Runnable r1 = new Runnable() {
        public void run() { System.out.println(this); } // 指向匿名类对象
    };
    Runnable r2 = () -> System.out.println(this); // 指向外部ThisDemo实例
}

Stream + Lambda · 业务实战演示

行为参数化典范

Stream API结合Lambda能对集合进行流水线操作，提升代码可读性与效率。

📊 复杂数据处理案例

// 员工列表：按薪资过滤、排序、提取姓名
List<Employee> employees = getEmpList();
List<String> highPaidNames = employees.stream()
    .filter(e -> e.getSalary() > 8000) // Predicate
    .sorted(Comparator.comparing(Employee::getSalary).reversed())
    .map(Employee::getName) // Function
    .limit(5)
    .collect(Collectors.toList());

// 使用reduce聚合计算 总和
int total = employees.stream()
    .map(Employee::getSalary)
    .reduce(0, Integer::sum);

🧪 函数式接口串联

// 使用Consumer + Predicate 链式处理
Predicate<String> notEmpty = s -> s != null && !s.trim().isEmpty();
Consumer<String> printer = s -> System.out.println(">> " + s);

List.of("Java", "", "Lambda", null).stream()
    .filter(notEmpty)
    .forEach(printer);

// 自定义函数式接口组合
Calculator addThenSquare = (a,b) -> (a+b)*(a+b);
System.out.println(addThenSquare.compute(2,3)); // 输出25

📌 使用前提

Lambda表达式必须出现在期望函数式接口实例的上下文中（赋值、方法参数、转型）才能被正确编译。

Runnable r = () -> {} ✅
Object o = () -> {} ❌ 需要强转
🧠 变量捕获细节

Lambda中引用的局部变量必须是 effectively final（未重新赋值），实例变量和静态变量无此限制。

int count = 1;
Runnable r = () -> System.out.println(count);
// count = 2; 再编译会报错
⚙️ 延迟执行 & 惰性求值

Stream 中间操作 (filter/map) 是惰性的，只有在遇到终端操作 (forEach/collect) 时才执行Lambda，极致性能优化。

parallelStream 配合Lambda需要注意线程安全
