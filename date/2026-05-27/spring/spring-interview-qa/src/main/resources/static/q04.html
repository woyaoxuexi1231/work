📋 Q04 · FactoryBean

名字很像但完全不是一回事——FactoryBean 是 Bean，BeanFactory 是容器。 · 🔬 代理生成动画 → · 🧬 纯接口→对象 全链路 → · 🗄️ 方法→SQL 完整映射 →

面
FactoryBean 和 BeanFactory 是一回事吗？
我
不是。记住一个口诀：

BeanFactory 以 Factory 结尾——它是工厂（Spring IOC 容器的顶层接口，管理所有 Bean 的创建和获取）。
FactoryBean 以 Bean 结尾——它本身是一个 Bean（一个能定制 Bean 创建逻辑的特殊 Bean）。

最大的区别：getBean("xxx") 返回的是 FactoryBean.getObject() 的产物，不是 FactoryBean 本身。想拿 FactoryBean 本身？加 & 前缀：getBean("&xxx")。
面
MyBatis 的 Mapper 接口是怎么通过 FactoryBean 生成代理的？
我
MyBatis-Spring 为每个 Mapper 接口注册一个 MapperFactoryBean，它的 getObject() 返回 JDK 动态代理。调用 mapper.findById(1L) 时 → 代理拦截 → 拼 SQL → 执行 → 返回结果。

为什么用 JDK 代理？因为 Mapper 是接口，没有实现类——JDK 代理天然适合接口代理。CGLIB 需要实现类作为父类。而且 Mapper 代理是无状态的——真正的数据库连接由 SqlSessionTemplate 通过 ThreadLocal 管理，所以一个代理实例能被所有线程安全共享。
面
getObject() 返回的对象能被 AOP 增强吗？
我
能。getObject() 返回的对象会经过完整的 BeanPostProcessor 链——包括 AOP 代理生成。这是 Spring 对 FactoryBean 的特殊处理：它知道用户关心的是产物，不是 FactoryBean 自身。
追问："FactoryBean 的 isSingleton() 默认返回 true，有什么坑？"
答：即使你的 getObject() 每次返回新对象，Spring 也只会调用一次并缓存结果。如果你需要每次返回新实例，必须重写 isSingleton() 返回 false。
❓ 自问：除了 MyBatis，还有哪个框架用 FactoryBean？
答：Feign 的 FeignClientFactoryBean——和 MyBatis 同样的思路，为 @FeignClient 接口生成 HTTP 代理。Hibernate 的 SessionFactory 也通过 LocalSessionFactoryBean 创建。FactoryBean 模式在"接口代理"场景下是标准做法。
