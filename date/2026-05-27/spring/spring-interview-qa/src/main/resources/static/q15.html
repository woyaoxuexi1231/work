📋 Q15 · 分布式事务 — 不用 Seata，Spring 怎么搞分布式事务？

面
只用 Spring 框架，不引入 Seata，你能怎么实现简单的分布式事务？
我
三种方案——

1. TCC 手动实现：Try 冻结资源 → Confirm 真正扣减 → Cancel 释放冻结。挑战：空回滚（Cancel 时 Try 可能没执行）、悬挂（Cancel 比 Try 先到）、幂等（可能重复调用）。都需要事务 ID 去重。
2. 本地消息表：业务 + 消息写入同一本地事务 → 定时任务扫描发送 → 消费方幂等。最终一致性方案，不需要全局事务管理器。
3. JTA：配置 Atomikos + XA 数据源，用现有 @Transactional。但 XA 协议同步阻塞不适合高并发。

Spring 为什么没内置？分布式事务是一个问题空间而非解决方案——CAP 约束下没有银弹。Spring 的选择是提供 PlatformTransactionManager SPI，让 JTA/Seata 等作为插件接入。
❓ 自问：什么时候用 TCC vs 本地消息表？
答：需要强一致性 + 实时扣减（如库存）→ TCC；能接受短暂延迟 → 本地消息表更简单。
