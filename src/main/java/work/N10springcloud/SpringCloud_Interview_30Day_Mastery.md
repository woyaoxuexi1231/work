# Spring Cloud 30天面试突击指南 (🌐 分布式架构核心版)

> **总纲**：本指南深度聚焦于 **Spring Cloud (Standard/Netflix/Common)** 体系。我们将从分布式理论（CAP/BASE）出发，全面攻克服务治理、负载均衡、声明式调用、熔断限流、网关路由、分布式配置及链路追踪。
> 
> **注意**：本指南不包含 Spring Cloud Alibaba 特有组件（如 Nacos, Sentinel），仅聚焦于 Spring Cloud 官方维护/集成的通用组件及 Netflix 经典组件。
> 
> **学习路径**：
> 1. **Week 1 (1-7天)**：微服务理论基础与服务治理 (Eureka/Consul/Zookeeper)。
> 2. **Week 2 (8-14天)**：客户端负载均衡 (Ribbon/LoadBalancer) 与声明式调用 (Feign/OpenFeign)。
> 3. **Week 3 (15-21天)**：服务容错 (Hystrix/Resilience4j)、分布式配置 (Config) 与 API 网关 (Gateway)。
> 4. **Week 4 (22-26天)**：链路追踪 (Sleuth/Zipkin)、消息驱动 (Stream) 与分布式安全 (Security)。
> 5. **Week 5 (27-30天)**：分布式事务理论、架构演进、性能调优与大厂全链路实战。

---
