## 二、逐题血淋淋复盘

### 面试题 1：synchronized 加在静态方法和实例方法上有什么区别？

- **你说的（更新版）**：加了 ACC_SYNCHRONIZED 标记和等价写法 `synchronized(Test.class)` / `synchronized(this)`。
- **得分**：12→**15/20** ✅  
- **哪里对了**：**补了上次缺的两个关键点，方向正确。**
  - `javap -v` 能看到 `ACC_SYNCHRONIZED`——对了，说明你真的去 `javap` 看了。
  - 等价于 `synchronized(Test.class)` 和 `synchronized(this)`——也对了。
  - 从 12 分到 15 分，这 3 分是给你"反馈后行动"的奖励分。
- **哪里错了 / 哪里不够**：
  - • **只说"多了一个 ACC_SYNCHRONIZED"，但没有对比**：同步方法 vs 同步代码块的字节码差异是什么？同步方法靠的是 **方法级 ACC_SYNCHRONIZED 标志**，JVM 执行方法时自动获取释放管程；同步代码块靠的是 **monitorenter / monitorexit 指令**。你没说这个对比。
  - • **错别字传承**：原答案写的是"**加载实例方法**"，新答案写的是"**加载静态方法和示例方法**"——"加载"应该是"加在"，"示例"应该是"实例"。从原答案里把错别字也复制过来了，看起来像是直接粘贴了原文改了几个字。
  - • **ACC_SYNCHRONIZED 的进一步理解空缺**：如果你能再说一句"ACC_SYNCHRONIZED 是 JVM 规范定义的，HotSpot 在执行该方法时会隐式调用 `ObjectMonitor::enter`/`exit`，与 monitorenter 指令在运行时的行为一致"，面试官会眼前一亮。
- **你该补什么（知识点地图）**：
  - 去查：javap -v 分别反编译一个同步方法和一个同步代码块，对比两者的字节码结构
  - 去练：讲清楚"方法级的 ACC_SYNCHRONIZED 是 JVM 执行引擎层面的隐式同步，代码级的 monitorenter/exit 是字节码指令层面的显式同步"
- **最后一句真话**：15 分，进步了但没满。你把缺的点补了，但没有展开对比。**下次如果能做"同步方法 vs 同步代码块"的字节码对比，18+ 稳拿。**
