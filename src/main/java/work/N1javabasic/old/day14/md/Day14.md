## 题目 A：手写 LRU Cache（带过期时间）

### 原理回顾题

#### HashMap 扰动函数

因为哈希表的大小一般很难超过 16 位整数，所以哈希码高16位很难参与哈希碰撞计算。

这里通过扰动函数，把高16位和第16位进行或与运算，使得高16位也能参与哈希碰撞得计算，这样可以使得分布更均匀。

如果不用扰动函数，在数组长度为 16 时，hash 值为 `0000 0000 0000 1111` 和 `0000 0000 1111 1111` 的两个 key 会映射到同一个桶吗？ 是的

#### CopyOnWriteArrayList 缺陷

为什么它的迭代器是"弱一致性"的？

因为是写时复制的，在写的过程中并不影响之前的读操作，但是读操作读取的旧的数据，读操作并不能获取到最新的数据，所以存在一个弱一致性的问题



如果在遍历过程中另一个线程添加了元素，迭代器会报错吗？

不会报错，读线程和写线程压根就没有操作同一个数组。写线程会复制一份数据出来，再进行操作，操作完成之后再进行替换，把原来的旧的数组替换新的数组，所以对读的线程没有任何影响。

## 题目 B：实现令牌桶限流器

编程题有点难。

### 原理回顾题

#### `volatile` 写操作会插入 `StoreStore` 和 `StoreLoad` 屏障，这两个屏障分别阻止了什么类型的重排序？

StoreStore 会插入被volatile修饰的对象的写操作之前，此写操作之前的所有写操作都必须完成才会进行此写操作

StoreLoad 会插入被volatile修饰的对象的写操作之后，确保此写操作之后的读操作能够读取到此写操作及之前所有的写操作的最新值



#### `ReentrantLock` 的 `unlock()` 是如何唤醒等待队列中的下一个线程的？

1. 首先他会会把aqs的state状态为0，表示自己已经不再持有锁
2. 然后他找到自己的下一个节点是否需要唤醒，如果找不到，那么从尾部开始从后往前找，找到最前面的一个需要唤醒的线程，通过locksupport.unpark进行唤醒

#### 如果唤醒失败（线程被中断），会发生什么？

这会导致后续节点无法被唤醒了，之后后续有线程进来加锁后再次唤醒才能使之前的节点被唤醒。



#### 线程池的 `Worker` 类为什么继承 `AQS` 而不是直接用 `ReentrantLock`？

继承AQS可以自己定义吧，本来加锁操作都是比较轻的这里，使用reentrantlock就比较重了。
