#### String 为什么要设计成不可变的？从安全和性能两方面回答。

**安全方面**

1. 线程安全，因为不可变的特性，多线程使用的时候是天然安全的
2. 网络安全，不会受到可变因素而导致的网络攻击

**性能方面**

1. 字符串的不可变，非常适合做hashmap的key键
2. 字符串池化，可以节省大量的内存



#### 谈谈泛型擦除。既然擦除了，为什么 Java 还能在运行时通过反射获取泛型信息？

1. 泛型擦除最主要的目的是为了兼容jdk1.5之前的代码，擦除后的类型为Object，这样之前的代码也能够在高版本的jdk环境中运行
2. 擦除后可以通过 ParamType?还是叫啥的 类获取泛型定义是给定的泛型信息，这是jvm放入类信息中的。但是分为了 运行时擦除和编译时擦除 ，运行时擦除是不可以的，因为在编译时就没有明确定义到底时什么类型，通过反射获取到也仅仅是编码时定义的泛型信息，并不是一个直接的类信息



#### 反射为什么慢？有哪些优化手段？

1. 反射要进行安全检查
2. 获取方法、属性的时候内部会进行复制，这样会带来性能损耗

响应的措施：

1. 对所有的属性直接设置 setAccesable为true

2. 缓存方法和属性

   

#### 动态代理在 Spring AOP 中是如何切换的？

spring aop 本身有 jdk 动态代理和 cglib 动态代理两种方式

对于有接口的类，默认采用jdk动态代理

对于没有接口的类，默认采用cglib动态代理

可以通过设置 proxy-target-class = true 来强制开启cglib动态代理



#### HashMap 1.7 的死循环是怎么产生的？1.8 是如何解决的？

死循环出现在扩容阶段，1.7扩容时在移动元素时采用头插法导致的死循环

1.8把头插法换成了尾插法，从而解决此问题

#### CopyOnWriteArrayList 的缺点是什么？适用于什么场景？

缺点：写时复制，那么意味着它的写操作非常耗内存，因为每一次写操作都需要复制一个新的数组出来，缺少强一致性

使用场景：写少读多的场景，不需要强一致性的场景



#### ArrayList 扩容是 1.5 倍，为什么不是 2 倍？

使用2倍扩容的时候，新申请的内存总是大于之前所释放的内存大小

1.5倍的时候，后续在扩容的时候需要新申请的内存大小是会低于之前释放的内存大小的，这样可以利用之前的内存碎片，从而提高内存利用率
