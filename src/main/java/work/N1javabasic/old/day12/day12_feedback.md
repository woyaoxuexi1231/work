## 二、逐题血淋淋复盘

### 面试题 1：线程池的执行流程是怎样的？先加队列还是先开线程？
- **你说的**：execute 提交，低于 corePoolSize 开 worker，满了进队列，队列满开到 max，最后拒绝。
- **得分**：18/20  
- **哪里对了**：
  - • **主链路清晰**：四个步骤描述得很准确，这是基础中的基础。
- **哪里错了 / 哪里不够**：
  - • **描述不够专业**：不要说“先开线程再开队列”，这会误导人。准确表述是：**“先填充核心线程，再填充阻塞队列，最后填充非核心线程”**。
  - • **面试官心想**：基础还行，但表达上缺乏一种“严谨感”。

### 面试题 2：如何根据业务类型（IO 密集型 vs CPU 密集型）配置核心线程数？
- **你说的**：举例 4 核 CPU，50% IO 则设 8 线程。
- **得分**：12/20  
- **哪里错了 / 哪里不够**：
  - • **太理想化**：面试官不想听你的“假定 50%”，他想听公式和经验值。
  - • **面试官心想**：典型的“书生算法”。
- **你该补什么**：
  - **CPU 密集型**：N（CPU 核心数）+ 1。为什么要 +1？（提示：为了应对偶尔的页缺失或其他导致线程暂停的意外）。
  - **IO 密集型**：2N 或 N / (1 - 阻塞系数)。
  - **终极答案**：所有公式都是参考，**生产环境必须经过压测（结合性能监控）来动态调整**。

### 面试题 3：线程池中的线程如果抛出了未捕获异常，会发生什么？
- **你说的**：线程终止，worker 移除，创建新 worker 补偿。
- **得分**：10/20  
- **哪里错了 / 哪里不够**：
  - • **致命缺失**：你只回答了 `execute()` 的情况。如果用 `submit()` 提交任务，**异常会被吞没**，除非你调用 `Future.get()`，否则控制台连个响声都没有！
  - • **面试官心想**：又是一个没写过 bug 的。
- **你该补什么**：
  - 去搜：`ThreadPoolExecutor.runWorker` 源码里对 `afterExecute` 的调用，以及 try-catch 的包裹逻辑。
