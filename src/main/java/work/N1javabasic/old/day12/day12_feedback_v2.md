## 二、逐题血淋淋复盘

### 面试题 1：线程池的执行流程是怎样的？先加队列还是先开线程？
- **你说的**：先填充核心线程，再填充阻塞队列，最后填充非核心线程。
- **得分**：20/20  
- **哪里对了**：
  - • **专业性显著提升**：这次的总结非常地道，面试官听到“填充”这个词就知道你真的理解了资源分配的优先级。
- **最后一句真话**：这题过关，保持这种严谨的表述。

### 面试题 2：如何根据业务类型（IO 密集型 vs CPU 密集型）配置核心线程数？
- **你说的**：上公式：2N 或 N/(1-阻塞系数)；CPU 密集型 N+1。
- **得分**：18/20  
- **哪里对了**：
  - • **深度到位**：对 +1 的解释（页缺失、硬件中断）非常加分，这体现了你对 OS 底层的认知。
- **哪里不够**：
  - • **缺少权衡**：面试官还会问你，如果 N+1 依然导致 CPU 飙升怎么办？（答案：限流或增加机器，而不是盲目调大线程数）。

### 面试题 3：线程池中的线程如果抛出了未捕获异常，会发生什么？
- **你说的**：execute 和 submit 异常时都不会有反应，只是 worker 被中断并新建。
- **得分**：14/20  
- **哪里错了 / 哪里不够**：
  - • **认知还是有偏差**：你说“都没有反应”是错的。
  - • **核心区别**：
    - `execute()`：异常会直接抛给 `Thread.UncaughtExceptionHandler`，默认会打印堆栈到 `System.err`。
    - `submit()`：**异常被吞得干干净净**。它被封装在 `FutureTask` 里的 `outcome` 字段中，只有当你调用 `future.get()` 时，它才会以 `ExecutionException` 的形式重新抛出。
  - • **面试官心想**：源码读了，但没读透 `FutureTask` 那一块。
