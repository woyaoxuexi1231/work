## 1.7 的死循环是怎么产生的？1.8 是如何解决的？

1.7的死循环是由于 hashmap产生哈希冲突的时候，多个线程在指定哈希桶的位置同时需要元素时，而产生的。

具体过程应该是  

线程a到达指定哈希桶的位置，发现冲突了，这个时候要进行头插法(1.7的设计)

线程b此时也到达哈希桶的位置，也发现冲突了，这个时候也要进行头插法

所以线程a先把 head改为插入元素，原head的头执行插入元素，再把head的尾指针改为原head，然后把插入元素放入到指定的哈希桶的位置

线程b也执行这个操作，但是呢，在线程a把元素  这里忘记了

## 扰动函数（hash 方法）的作用是什么？为什么数组长度要是 2 的幂次方？

扰动函数，第一次听这个说法，hash方法其实就是提供了一个随机的整数，整个整数只是为了证明两个对象想不想等。

数字长度是2的幂次方，主要是为了好扩容，进行左移时正好就是扩大2倍。
