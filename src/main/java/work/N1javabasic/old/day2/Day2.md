## 反射为什么慢？有哪些优化手段？

反射在运行时动态获取类信息，在通过类信息去详细操作对象的属性和方法。

这肯定比直接操作对象要慢得多，多了好几个步骤。

尽量不要使用反射，如果要使用反射我就不知道了。



## JDK 动态代理为什么要求必须有接口？CGLIB 是如何实现代理的？

jdk动态代理必须要求实现需要代理的对象的接口，如果对象没有接口是不行的

最后的对象是一个实现了目标对象和继承了proxy类的一个对象。

因为实现了目标对象的方法，在使用代理对象时，可以直接使用目标接口的方法。

cglib是直接通过生成字节码对象来实现代理，通过反射得到对象的信息(有哪些方法，方法入参是什么)，然后通过字节码生成代理对象实现相同的方法来代理目标对象。



## 动态代理在 Spring AOP 中是如何切换的？

SpringAop提供了一个参数，proxyclass = ture，开启强制通过cglib代理。
