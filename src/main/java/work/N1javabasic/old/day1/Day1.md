## 为什么重写 `equals` 必须重写 `hashCode`？不重写会有什么后果？

如果对象永远不会被放入集合类，那么可以不用重写。

集合类会使用hashcode和equals同时判断对象是否相等，如果只写了equals，会出现equals相等，但是hashcode不相等对象，两个相同的对象却被放入了不同的位置，而再用相同equals相等的对象来找，却大概率找不到，因为hashcode大概率不相等。



## `String s = new String("abc")` 到底创建了几个对象？

首先，堆中几乎存放了所有对象，但是内存中还有一块区域用于存放一些特殊的对象，这里就包括了这种特殊对象，那就是静态变量。 "abc" 这是一个字符串，同时他也是一个字面量，字面量"abc"会被放入静态变量池，方便重用。所以这里创建了几个对象要看静态变量池是否已经存在了 "abc" 这个对象，如果不存在，就创建了两个，如果已经存在，就创建了一个。



## `String` 为什么要设计成不可变的？从安全和性能两方面回答。

不仅仅是字符串，八大基础类型的封装类型，五个数字、两个浮点数和一个布尔，全部被设计成不可变对象。

从安全性来看，String作为基础类型，不可变是为了确保不管在什么情况下，拿到string的那一刻开始，这个值不会再任何情况下改变，不会引起前后的不一致性。

从性能更来看，不可变对象需要维护的代码量更少？不可变对象减少更多的代码判断？



## 谈谈泛型擦除。既然擦除了，为什么 Java 还能在运行时通过反射获取泛型信息？

为了在编译时使得集合类的使用更加安全，引入了泛型，而后泛型得到了更广泛的使用，泛型类，泛型方法。

总而言之，泛型时为了解决某一类集合对象能够更加安全的使用而引入的机制。

为什么要擦除？？

擦除后，Java只能通过得到泛型中确切的对象，才能对象获得泛型信息，而不能直接通过泛型类获得其对应的泛型 信息。
