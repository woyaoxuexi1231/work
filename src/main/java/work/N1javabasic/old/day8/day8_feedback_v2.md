## 二、面试官“犀利”讲评（针对拒绝改进项）

### 1. 关于 Happens-Before 的“记不清”
- **面试官视角**：在技术面试中，说“我不记得了”是可以接受的，但在**复盘练习**中，这代表你放弃了对知识盲区的探索。Happens-Before 是 JMM 的宪法，不掌握它，你永远无法理解 Java 的并发安全保障。
- **强制补课**：去查阅 **程序次序规则、管程锁定规则、volatile 变量规则**，并告诉我为什么它们能解决重排序带来的可见性问题。

### 2. 关于 count++ 的“count = count + 1”
- **面试官视角**：如果你在阿里三面中这样回答，面试官会直接结束面试。`count = count + 1` 本身也是一个非原子操作，你的回答是用一个错误解释另一个错误。
- **底层真相**：请去查看 `javap -c VolatileTest.class` 的输出，找到 `getstatic`, `iadd`, `putstatic` 这三条指令，并理解为什么线程 A 执行到 `iadd` 时，线程 B 读到的 `getstatic` 还是旧值。

### 3. 关于编程题的“结果验证”
- **面试官视角**：你在代码中注释说“volatile 没帮助”，但你**没有通过运行结果证明它没帮助**。
- **缺失动作**：在 main 函数末尾加上 `t1.join(); t2.join(); System.out.println(count);`。如果结果不是 20000，你的代码才真正具有说服力。
