# 计算机网络 30天面试突击指南 (🌐 互联网协议与底层内核专家版)

> **总纲**：本指南深度聚焦于 **计算机网络**。作为后端开发，网络是连接所有组件的纽带。我们将从 OSI 七层模型出发，深度攻克 TCP/UDP 传输层协议、HTTP 家族演进（1.1 -> 2.0 -> 3.0）、HTTPS 安全机制、DNS 解析原理、Linux 内核 IO 模型以及网络性能优化与故障排查。
> 
> **学习路径**：
> 1. **Week 1 (1-7天)**：网络模型、物理层/数据链路层基础与 TCP/UDP 传输层。
> 2. **Week 2 (8-14天)**：HTTP 协议详解：状态码、头部、缓存机制与演进。
> 3. **Week 3 (15-21天)**：HTTPS 安全机制：SSL/TLS 握手、证书、加密算法。
> 4. **Week 4 (22-26天)**：DNS、CDN、ARP/DHCP 与网络性能调优（内核参数）。
> 5. **Week 5 (27-30天)**：Socket 编程、Epoll 底层、分布式网络架构与故障排查实战。

---

## 📅 第一阶段：传输层核心与 TCP/UDP (筑基)

### 第1天：OSI 七层模型与 TCP/IP 四层模型

5. **地狱级**：如果要在物理层实现一个抗干扰的可靠传输，除了重传，你还能想到哪些底层编码技术（如 FEC）？

#### ### 编程题
编写一个简单的 Python 或 Java 程序，获取本机所有网卡的 IP 地址、子网掩码和 MAC 地址。

#### ### 编程题
