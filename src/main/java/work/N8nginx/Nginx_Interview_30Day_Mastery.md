# Nginx 30天面试突击指南 (🌐 高性能 Web 负载均衡专家版)

> **总纲**：本指南深度聚焦于 **Nginx**。我们将从其高性能的多进程事件驱动模型出发，全面攻克反向代理、负载均衡算法、动静分离、缓存优化、SSL/TLS 安全配置以及高可用架构。
> 
> **学习路径**：
> 1. **Week 1 (1-7天)**：核心架构、进程模型与事件驱动原理。
> 2. **Week 2 (8-14天)**：配置精讲：反向代理、负载均衡与动静分离。
> 3. **Week 3 (15-21天)**：性能调优：缓存、压缩、连接池与内核参数优化。
> 4. **Week 4 (22-26天)**：安全保障：SSL 优化、限流防攻击与 WAF 基础。
> 5. **Week 5 (27-30天)**：高可用架构：Keepalived 配合、Lua 扩展（OpenResty）与大厂实战。

---

## 📅 第一阶段：架构之基与事件驱动 (筑基)

### 第1天：Nginx 进程模型：Master-Worker 架构
#### ### 面试题
1. **基础**：Nginx 的多进程模型是如何工作的？（Master 与 Worker 的分工）。
2. **中级**：为什么 Nginx 使用多进程而不是多线程？（内存隔离、稳定性、CPU 亲和性）。
3. **高级**：Worker 进程的数量通常如何配置？为什么？
4. **源码**：Master 进程是如何管理 Worker 进程的？（信号量机制）。
5. **地狱级**：在不停止服务的情况下，Nginx 如何实现平滑升级（热部署）？请详细描述进程切换过程。

#### ### 编程题
编写一个 Shell 脚本，监控 Nginx Worker 进程的 CPU 和内存使用情况，并在异常时发送报警。

5. **地狱级**：什么是“零拷贝”在 Nginx 中的应用？`sendfile` 指令的底层原理是什么？
