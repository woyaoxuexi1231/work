# Redis 30天面试突击指南 (⚡ 极致性能与缓存艺术版)

> **总纲**：本指南深度聚焦于 **Redis**。我们将从核心数据结构出发，全面攻克 IO 多路复用模型、持久化机制、高可用架构（主从/哨兵/集群）、缓存雪崩/击穿/穿透、分布式锁以及底层源码实现。
> 
> **学习路径**：
> 1. **Week 1 (1-7天)**：核心数据结构底层实现与基础 IO 模型。
> 2. **Week 2 (8-14天)**：持久化 (RDB/AOF)、主从同步与哨兵架构。
> 3. **Week 3 (15-21天)**：Redis Cluster 集群、缓存策略与 Lua 脚本应用。
> 4. **Week 4 (22-26天)**：性能调优、内存管理、BigKey/HotKey 处理。
> 5. **Week 5 (27-30天)**：分布式锁、Redlock 算法、业务场景深度设计。
