# Redis 30天面试突击指南 (🛠️ 生产实战与场景驱动版)

> **总纲**：本指南拒绝纯理论堆砌，深度聚焦 **Redis 在生产环境中的真实应用**。我们将从业务痛点出发，攻克分布式锁的各种坑、缓存一致性的终极方案、海量数据的处理技巧以及线上事故的排查思路。
> 
> **学习路径**：
> 1. **Week 1 (1-7天)**：高并发业务组件实战（分布式锁、限流、幂等）。
> 2. **Week 2 (8-14天)**：复杂业务场景建模（排行榜、签到、附近的人、消息流）。
> 3. **Week 3 (15-21天)**：稳定性与一致性方案（三兄弟处理、双写一致性、延时双删）。
> 4. **Week 4 (22-26天)**：线上排障与性能调优（BigKey/HotKey 扫描、内存分析、慢查询）。
> 5. **Week 5 (27-30天)**：集群架构演进与全链路设计（混合持久化、跨机房同步、秒杀架构）。

---

## 📅 第一阶段：高并发业务组件实战 (筑基)

### 第1天：分布式锁 (基础) - 别让锁成了你的坑
#### ### 面试题
1. **基础**：在 Spring Boot 中，使用 `setnx` 实现简单锁时，如果忘记设置过期时间会发生什么？
2. **中级**：如何保证“判断锁是否存在”和“设置过期时间”这两个操作的原子性？
3. **高级**：锁过期了但业务没跑完，锁被误释放了怎么办？（看门狗机制）。
4. **源码**：Redisson 的 `tryLock` 内部是如何利用 Lua 脚本实现可重入性的？
5. **地狱级**：Redis 主从切换导致锁丢失（Master 挂了，Slave 还没同步锁就变成了 Master），除了 Redlock，你还有什么工程上的折中方案？

#### ### 编程题
使用 Spring Data Redis 实现一个基础的分布式锁工具类，包含加锁（带超时）、解锁（防误删逻辑）。
