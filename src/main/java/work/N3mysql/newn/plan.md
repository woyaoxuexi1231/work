## 第1天：SELECT + WHERE 基础过滤
本日掌握：最基本的单表查询，数值和字符串条件过滤；覆盖原理点：无；阶段：使用期

### 🟢 基础用法题
#### 题目1：查询特定会员的订单
**场景描述**：电商系统，会员表 `members` 包含 `member_id`, `name`, `city`, `birth_date`, `points` 字段。需找出所有积分大于500且所在城市为'北京'的会员。

**编码要求**：提供建表语句、插入至少5条测试数据，并编写一条使用 `WHERE` 条件过滤的查询。

**建表及数据**：
```sql
CREATE TABLE members (
  member_id INT PRIMARY KEY,
  name VARCHAR(50),
  city VARCHAR(20),
  birth_date DATE,
  points INT
);

INSERT INTO members VALUES
(1, '张三', '北京', '1990-05-20', 800),
(2, '李四', '上海', '1988-12-15', 300),
(3, '王五', '北京', '1995-07-01', 600),
(4, '赵六', '广州', '1992-03-30', 1200),
(5, '孙七', '北京', '1985-11-22', 450);
```

**查询**：积分大于500且城市为'北京'的会员：
```sql
SELECT * FROM members
WHERE points > 500 AND city = '北京';
```

**预期输出**：
```
member_id | name | city | birth_date | points
1         | 张三 | 北京 | 1990-05-20 | 800
3         | 王五 | 北京 | 1995-07-01 | 600
```

**🔍 原理反思提问**：如果 `city` 字段存有尾随空格，如 `'北京 '`，`city = '北京'` 还能匹配吗？为什么？

**💬 面试官可能追问**：`WHERE city = 'beijing'` 是否能查出来？MySQL 如何处理字符串比较的字符集与排序规则？

### 🟡 中级用法题
#### 题目2：多条件组合与 NULL 处理
**场景描述**：继续使用 `members` 表，业务需找出：积分小于1000且出生日期在1990年以后的会员，或者城市为空的会员（因为信息不完整需要跟进）。

**编码要求**：编写一条查询，使用 `OR` 和 `AND` 的组合，注意 `NULL` 的判断。

**查询**：
```sql
SELECT name, city, points
FROM members
WHERE (points < 1000 AND birth_date > '1990-01-01')
   OR city IS NULL;
```

在现有数据中插入一条城市为 NULL 的记录：
```sql
INSERT INTO members VALUES (6, '周八', NULL, '1993-08-08', 200);
```

**预期结果**：
```
张三 | 北京 | 800   -- points<1000 且 birth_date>1990-01-01
王五 | 北京 | 600   -- 同上
周八 | NULL | 200   -- city IS NULL
```

**🔍 原理反思提问**：如果用 `city = NULL` 替换 `city IS NULL` 结果会发生什么？为什么？

**💬 面试官可能追问**：AND 和 OR 的优先级是怎样？如何用括号确保正确逻辑？

### 🔴 高级用法题
#### 题目3：LIKE 模糊匹配与转义
**场景描述**：营销部门需要一份名单，所有姓名中包含字母 'F' 或下划线 `_` 的会员（因为系统旧接口遗留了一些含特殊字符的名字）。测试数据需包含 `F_name` 和 `A_B` 这样的名字。

**编码要求**：使用 `LIKE` 和 `ESCAPE` 处理下划线这种通配符。

**新增测试数据**：
```sql
INSERT INTO members VALUES
(7, 'F_name', '深圳', '1998-01-01', 100),
(8, 'A_B', '成都', '1999-06-06', 300);
```

**查询**：名字包含字母 `F` 或实际下划线的会员：
```sql
SELECT name FROM members
WHERE name LIKE '%F%' OR name LIKE '%!_%' ESCAPE '!';
```
说明：`%!_%` 中的 `!` 是转义符，让下划线被当作普通字符。

**输出**：
```
name
F_name
A_B
```

**🔍 原理反思提问**：如果不使用 `ESCAPE`，`LIKE '%_%'` 匹配什么？它和 `LIKE '%!_%' ESCAPE '!'` 有何不同？

**💬 面试官可能追问**：如何查找名字以 `%` 开头的会员？`LIKE '%\%%'` 合法吗？

### 🏢 大厂面试场景实战（使用层面）
**场景**：运营同事导出了一份 10 万行的 CSV，导入到 `user_import` 表中，其中 `phone` 字段有些包含中划线、空格，有些为空。要求你写一条 SQL 清洗出有效的手机号（纯数字，11位，不是 NULL 且不是空字符串）。

### 🔴 高级用法题
#### 题目3：分页包装与行号生成（不依赖窗口函数）
**场景描述**：在不支持窗口函数的旧版本 MySQL 中，需要给查询结果加上行号，并据此分页。使用用户变量模拟。

**查询**：
```sql
SET @row_number = 0;
SELECT row_num, name, points FROM (
  SELECT (@row_number := @row_number + 1) AS row_num, name, points
  FROM members
  ORDER BY points DESC
) AS t
WHERE row_num BETWEEN 3 AND 4;
```

**输出**：显示第3、4行。
