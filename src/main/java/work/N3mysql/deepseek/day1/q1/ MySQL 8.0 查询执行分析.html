关键结论

查询执行阶段分析

执行阶段(1.5-3.0ms)是最耗时的部分，占总执行时间的60-80%，而解析(0.2-0.5ms)和优化(0.1-0.3ms)阶段相对较快。

优化器行为分析

优化器在处理简单主键查询时会生成多种等价计划，最终选择成本最低的const访问方式。

大规模执行瓶颈

执行10万次相同查询时，执行阶段是绝对的性能瓶颈，占比高达98%。

查询缓存移除原因

基于严重的技术缺陷，包括全局锁争用、低效的缓存失效机制、内存管理问题等。
