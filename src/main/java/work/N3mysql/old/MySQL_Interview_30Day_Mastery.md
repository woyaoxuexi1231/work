# MySQL 30天面试突击指南 (🗄️ 数据基石与调优艺术版)

> **总纲**：本指南深度聚焦于 **MySQL**。我们将从存储引擎底层出发，全面攻克 B+ 树索引原理、事务 ACID 与隔离级别（MVCC）、锁机制、日志系统（Redo/Undo/Binlog）、查询优化、主从复制以及分库分表高阶架构。
> 
> **学习路径**：
> 1. **Week 1 (1-7天)**：存储引擎、数据类型与 B+ 树索引底层。
> 2. **Week 2 (8-14天)**：SQL 优化、执行计划（EXPLAIN）与索引失效分析。
> 3. **Week 3 (15-21天)**：事务、MVCC 原理、锁机制与死锁排查。
> 4. **Week 4 (22-26天)**：日志系统、Buffer Pool 缓冲池与主从同步原理。
> 5. **Week 5 (27-30天)**：分库分表（Sharding）、高可用架构与生产级性能调优。
