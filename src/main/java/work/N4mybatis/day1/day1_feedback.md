## 二、逐题精讲

### 面试题 1：MyBatis 的核心组件有哪些？
- **你的答案摘要**：列举了 Builder, Factory, Session, Executor 及其作用。
- **评分**：20/20  
- **可吸收的标准答案**：
  > 1. **SqlSessionFactoryBuilder**：根据配置信息构建 SqlSessionFactory，构建完即可销毁。
  > 2. **SqlSessionFactory**：创建 SqlSession 的工厂，生命周期等同于应用的生命周期。
  > 3. **SqlSession**：执行命令、获取 Mapper 的顶层接口，包含 SQL 执行的所有方法。**注意：它是线程不安全的，用完必须关闭。**
  > 4. **Executor**：MyBatis 的执行器，负责 SQL 语句的生成和查询缓存的维护。

### 面试题 2：为什么说 MyBatis 是“半自动”的 ORM？
- **你的答案摘要**：全自动自动生成 SQL，MyBatis 提供模板，更灵活。
- **评分**：18/20  
- **扣分明细**：
  - 细节缺失：未提到 Hibernate 对复杂 SQL 优化的无力感，以及 MyBatis 的 SQL 维护成本。
- **可吸收的标准答案**：
  > - **半自动**：MyBatis 不会自动生成 SQL，而是由程序员手写 SQL。它自动完成的是**参数映射**和**结果集映射**。
  > - **优点**：灵活性极高，支持原生 SQL，易于进行 SQL 性能优化；学习成本相对 Hibernate 较低。
  > - **缺点**：手写 SQL 工作量大；数据库移植性较差（不同数据库 SQL 语法略有差异）。

### 面试题 3：MyBatis 是如何实现接口绑定（Mapper 接口）的？
- **你的答案摘要**：通过 xml 的 namespace 和 sql 标签的 id 绑定。
- **评分**：15/20  
- **扣分明细**：
  - **致命缺失**：未提及 **JDK 动态代理 (Proxy)**。面试官最想听的就是这个词。
- **可吸收的标准答案**：
  > MyBatis 使用 **JDK 动态代理** 为 Mapper 接口创建代理对象。
  > 1. 当调用接口方法时，会被代理类（`MapperProxy`）拦截。
  > 2. 代理类根据“全类名 + 方法名”寻找匹配的 `MappedStatement`。
  > 3. 最后通过 `SqlSession` 执行底层的数据库操作。

### 面试题 4：Configuration 对象扮演什么角色？
- **你的答案摘要**：配置连接、XML、接口扫描。
- **评分**：18/20  
- **可吸收的标准答案**：
  > `Configuration` 是 MyBatis 的**全局配置中心**，采用**单例模式**。它解析并存储了：
  > 1. 数据源、事务管理器、类型处理器（TypeHandler）。
  > 2. 所有的 `MappedStatement`（即 SQL 映射信息）。
  > 3. 所有的缓存配置。
  > 它是整个 MyBatis 运行时的“大脑”。

### 面试题 5：海量数据操作下的性能瓶颈？
- **你的答案摘要**：猜是参数解析和结果映射。
- **评分**：15/20  
- **扣分明细**：
  - 描述不够全面：未提及反射开销的具体体现。
- **可吸收的标准答案**：
  > 瓶颈主要出现在 **结果集映射 (ResultSet Mapping)** 阶段。
  > 1. **反射开销**：MyBatis 需要通过反射将数据库字段赋值给 Java 对象，字段越多，开销越大。
  > 2. **对象创建**：海量数据意味着创建百万级、千万级的 Java 对象，会给 JVM 带来巨大的 **GC 压力**。
  > 3. **SQL 解析**：如果 SQL 极其复杂且包含大量动态标签，参数解析也会有一定开销。

## 三、编程题点评

**原题**：创建一个基础的 MyBatis 项目（不依赖 Spring），编写一个 SqlSessionFactory 的初始化代码，并手动开启 SqlSession 执行一条 SQL。

**你的代码分析**：
- [TestController.java](file:///d:/project/demo/demo-java/work/src/main/java/work/N4mybatis/day1/TestController.java): 虽然写在 Spring 工程里，但逻辑完全手动，符合“不依赖 Spring 自动装配”的要求。
- [mybatis-config.xml](file:///d:/project/demo/demo-java/work/src/main/resources/day1/mybatis-config.xml): 标准的配置方式，包含环境和 Mapper 注册。

#### 1. 正确性验证
- [x] 手动加载 XML：使用 `Resources.getResourceAsStream`。
- [x] 手动构建 Factory：`new SqlSessionFactoryBuilder().build()`。
- [x] 正确关闭资源：使用了 `try-with-resources`。

#### 2. 代码风格与工程原则
- **好的地方**：
  - `try-with-resources` 的使用非常专业。
  - XML 配置结构清晰。
- **改进建议**：
  - 建议在 Mapper 接口上增加 `@Mapper` 注解（虽然原生 MyBatis 不需要，但在整合 Spring 环境下更规范）。
  - `String resource = "day1/mybatis-config.xml";` 这种硬编码路径在大型项目中建议放入常量池。

#### 3. 鲁棒性检查
- [x] 处理了文件未找到异常。
- [x] 处理了 SQL 执行异常。
